# NoCapEdit (nce)

NoCapEdit は、起動直後からすぐに書き始められ、ファイル名を考えずに保存できることを重視した、シンプルで洗練されたテキストエディタです。

一般的なテキストエディタで発生する「保存時にファイル名や保存場所を指定する」という手間をなくし、書くことに100%集中できるシームレスな体験を提供します。

---

## 🚀 主な特徴

1. **起動時・タブ作成時のファイル自動生成**
   - アプリの起動時、またはタブバーの `+` ボタンを押した瞬間に、設定されたホームフォルダ配下に `YYYYMMDD_HHMMSS.nctx` 形式のテキストファイルが自動生成されます。
   - 同名ファイルがすでに存在する場合は、自動的に `_01`, `_02` のように連番が付加され、上書きや競合を回避します。

2. **シームレスな自動保存 (Debounce 3秒)**
   - 文字の入力が停止してから **3秒後** に自動的にファイルを保存します。
   - 保存状態の更新はステータスバー左下のテキスト（「編集中...」「保存済み」）のみで行われ、ポップアップや通知など作業の集中を妨げる表示は一切行われません。

3. **空白ファイルの自動クリーンアップ**
   - 起動時や `+` ボタンで自動生成されたタブで、文字を何も入力しないまま（半角/全角スペース、改行、タブのみの「見えない文字のみ」の状態）でタブを閉じるか、アプリを終了した場合、ディスク上に作成されたファイルは**自動的に削除**されます。これにより、不要な空ファイルがフォルダに溜まりません。

4. **安全保存方式と保存ロック（排他制御）**
   - ファイル保存時は、一度一時ファイル (`.tmp`) に書き出してからリネーム置換する安全な方式を採用しています。OneDrive 等の同期フォルダ内でのファイルロック競合による破損を防ぎます。
   - 自動保存中にタブの切り替えやアプリ終了が重なった場合でも、同じファイルに対して同時に書き込みが走らないよう、タブごとに非同期処理の排他制御ロック（`isSaving`）を導入しています。

5. **テーマ切り替えとフォントカスタマイズ**
   - ダークモードとライトモードの切り替えに対応（Windows のタイトルバー色も連動）。
   - `Ctrl + マウスホイール` または `Ctrl + "+"` / `"-"` によるフォントの拡大縮小機能。
   - 等幅フォント（Monospace）とその他フォントを分類して選択できるフォントファミリー切り替え機能。
   - 設定はアプリ再起動後も維持されます。

6. **タイトルバーへのバージョン表示**
   - アプリケーションのタイトルバーに現在のバージョン番号が `NoCapEdit [ Ver 0.1.5 ]` の形式で表示され、現在のバージョンをひと目で確認できます。

## 🛠️ 動作環境とビルド・開発方法

初期 MVP は Windows 環境を対象として構成されています。

### 前提条件
- Rust 1.70 以上
- Visual Studio Build Tools (Windows)
- Node.js (フロントエンドのビルドおよびパッケージ管理)
- Tauri CLI (`cargo install tauri-cli`)

### 開発・起動手順
ターミナル（PowerShell等）でプロジェクトルートディレクトリを開き、以下のコマンドを実行します。

```powershell
# アプリケーションのビルドと起動
cargo run
```
*(※ Tauri CLIがインストールされている場合は `cargo tauri dev` でも起動可能です)*

### 配布パッケージのビルド
インストーラーをビルドする場合は以下のコマンドを実行します。

```powershell
cargo tauri build
```

また、ポータブル版（ZIP形式）を作成する場合は、以下のコマンドを実行して `target/release/bundle` 配下に ZIP アーカイブを出力します。

```powershell
# ポータブル版のビルドとZIPアーカイブの作成
cargo build --release
New-Item -ItemType Directory -Force -Path "target/release/bundle"
Compress-Archive -Path "target/release/NoCapEdit.exe" -DestinationPath "target/release/bundle/NoCapEdit_v0.1.5_x64_portable.zip" -Force
```

## 📁 データと設定の保存場所

- **設定ファイル**: `%APPDATA%/NoCapEdit/config.json`
- **デフォルトの保存先 (ホームフォルダ)**: `C:/Users/<ユーザー名>/Documents/nce`
  - 初回起動時、または設定されたフォルダが見つからない場合は、起動時にホームフォルダ選択ダイアログが表示されます。
