# 設定値・定数管理のリファクタリング（ハイブリッドアプローチ）

## 目標
ハードコードされ、コードベース全体に散在している設定値やマジックナンバーを、Rust、CSS変数、JSそれぞれに適切に分離・集約することで保守性と拡張性を高める。

## 背景
現状では、デフォルトの保存先パスやフォント指定、初期設定値などが Rust 側と JS 側で二重管理されている。また、一部の UI の描画値が JS 側でハードコードされており、CSS と責務が混在している。これを解消する。

## 実装計画

### 1. 基本設定・振る舞い（Rust 側への集約）
* **内容**: `src/main.rs` 内に設定管理用の定数定義（または別モジュール）を作成する。
* **対象値**: 
  * `SINGLE_INSTANCE_PORT` (49423) およびホスト `"127.0.0.1"`
  * ウィンドウのデフォルトサイズ (900x600, 400x300)
  * `APP_NAME` ("NoCapEdit"), 拡張子 (`.nctx`), 日付フォーマット
  * デフォルトの設定値（フォントサイズ、テーマ、行間など）
* **JS連携**: JS（フロントエンド）は独自のデフォルト値（保存先パスやフォント等）をハードコードせず、起動時の `invoke('get_settings')` 等を通じて初期値を取得する形に改める。

### 2. デザイン・UI設定（CSS変数への集約）
* **内容**: `src/dist/style.css` の `:root` にエディタ表示用の変数を追加する。
* **対象値**: 
  * フォントファミリの定義: `--editor-font-family: 'Fira Code', 'Monaco', 'Menlo', monospace;`
  * フォントサイズ: `--editor-font-size: 13px;`
  * 行間: `--editor-line-height: 1.5;`
* **JS連携**: `main.js` からインラインのスタイル（`element.style.fontFamily` 等）を直接書き換えるのではなく、`document.documentElement.style.setProperty` を使って上記のCSS変数を操作するように修正する。

### 3. フロントエンド固有ロジック（JS側の定数化）
* **内容**: `src/dist/main.js` の先頭部分に、フロントエンドでのみ使用される定数をまとめる。
* **対象値**: 
  * `AUTOSAVE_DELAY_MS = 3000`（既存）
  * タブ等のDOMに関する静的な文字列やステータスのメッセージ文字列など。

## 確認方法
- アプリをビルドして起動し、エラーなく画面が表示されること。
- テーマ変更、フォント変更、行間や文字サイズのズーム機能が従来通り機能すること。
- 自動ファイル生成の名称や保存時のパスが正しいこと。
