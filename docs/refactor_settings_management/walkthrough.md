# 設定値・定数管理のリファクタリング 実装確認

## 概要
実装計画に基づいて、設定値と定数の一元管理（ハイブリッドアプローチ）のリファクタリングを完了しました。コードベース全体に散在していたマジックナンバーを整理し、保守性と拡張性を高めました。

## 変更内容

### 1. 基本設定・振る舞い（Rust 側への集約）
- `src/main.rs` の先頭に定数定義を追加し、ポート番号（`49423`）、ホスト（`127.0.0.1`）、ウィンドウサイズ（`900x600`等）、デフォルトのテーマやフォントなどの設定値を一元管理しました。
- アプリ内のファイル名生成やフォルダ名取得ロジックでこれらの定数を使用するように修正しました。

### 2. デザイン・UI設定（CSS変数への集約）
- `src/dist/style.css` の `:root` にエディタ表示用のCSS変数（`--editor-font-family`, `--editor-font-size`, `--editor-line-height`）を追加しました。
- `.editor` クラスのスタイル指定を、ハードコードからCSS変数の参照に変更しました。

### 3. フロントエンド固有ロジック（JS側の定数化）
- `src/dist/main.js` から、`appState` 宣言時のハードコードされた初期値（`theme: 'dark'` 等）を削除し、Rustからの設定取得時まで未設定状態（`null`）とするようにしました。
- ファイル先頭に、`AUTO_FILE_REGEX`（自動保存ファイルの正規表現）、`FILE_EXT_NCTX` などのフロントエンド固有の定数をまとめました。
- フォントや文字サイズ、行間の変更時（`applyFontFamily`, `applyFontSize` 等）に、インラインスタイルを直接書き換えるのではなく、`document.documentElement.style.setProperty` を用いてCSS変数を更新する方式に変更しました。
