# v0.2系 タブ移動時のエディタインデント挿入不具合の修正計画

v0.2系において、エディタにフォーカスがある状態で `Ctrl + Tab` または `Ctrl + Shift + Tab` を押してタブを切り替える際、エディタに Tab 文字（インデント）が挿入されてしまう不具合を修正します。

---

## 🔍 不具合の概要と原因
* **現象**: 複数タブを開いている状態でエディタにフォーカスがある際、`Ctrl + Tab` などのショートカットでタブを切り替えると、タブは切り替わりますが、元のエディタ領域に Tab 文字が1文字挿入されてしまいます。
* **原因**: [js/main.js](file:///c:/work/NoCapEdit/src/dist/js/main.js) 内の `elements.editor.addEventListener('keydown')` の中で Tab キーのインデント挿入処理を行っていますが、`Ctrl` キーや `Alt` キーが同時に押されているかどうかのチェックが存在しないため、ショートカットキー押下時にもインデント文字が挿入されてしまっています。

---

## 🛠️ 修正内容

### フロントエンド (JavaScript)

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/js/main.js)
* エディタの `keydown` リスナーにおいて、`Tab` キー押下判定の直後に `Ctrl` または `Alt` キーが押されているかをチェックし、押されている場合はインデント処理をバイパス（何もしないで `return`）するガード条件を追加します。

```javascript
    elements.editor.addEventListener('keydown', (e) => {
        if (e.key === 'Tab') {
            // CtrlキーやAltキーが同時に押されている場合は、タブ移動などのショートカットとして処理するため、ここでは無視する
            if (e.ctrlKey || e.altKey) {
                return;
            }

            e.preventDefault();
            ...
```

### 2. 動作確認（手動検証）
ユーザーによる動作確認をお願いします。
* `cargo tauri dev` でアプリを起動し、テキストエリアにフォーカスを当てた状態で以下を確認します。
  1. `Ctrl + Tab` を押した際、タブが右隣に切り替わり、エディタに Tab 文字が**入力されない**こと。
  2. `Ctrl + Shift + Tab` を押した際、タブが左隣に切り替わり、エディタに Tab 文字が**入力されない**こと。
  3. 単純に `Tab` キーを押した際は、設定に応じたインデントが正しく挿入されること。
