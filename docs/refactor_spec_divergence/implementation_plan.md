# 仕様と実装の乖離修正 3-1: ステータスバー初期文言の修正（定数化対応）

## 1. 背景と目標
`docs/spec.md` のセクション4.5において、「曖昧な『準備完了』表示は行わない」と規定されていますが、実装コード（`main.js`, `settings.js`）内に `updateStatus('準備完了');` というハードコーディングが残存しています。
また、他のファイル（`tabs.js` 等）でも日本語テキストが直接ベタ書きされており、プロジェクトの「多言語化対応 (`i18n.js` の活用)」ルールから外れています。
本タスクでは、禁止文言の修正とあわせて、UI上の初期ステータス文言を `i18n.js` に定数として抽出し、それを参照する形へリファクタリングします。

## 2. 仕様に準拠した表示内容（i18n.js への追加）
`src/dist/i18n.js` の `DICT.ja` に以下のキーと値を新設します。
- `status_ready_auto`: `"保存準備完了"`
- `status_ready_manual`: `"※Ctrl+Sで保存できます"`
（※手動保存モード時の `[手動保存モード] ` プレフィックスは `updateStatus` 関数の共通ロジックによって自動付与されるため、多言語定数側には含めずDRY原則を保ちます）

## 3. 対象ファイルと修正内容

### [MODIFY] `src/dist/i18n.js`
- 上記のキーを辞書オブジェクトに追加します。

### [MODIFY] `src/dist/js/main.js` および `src/dist/js/ui/settings.js`
- `updateStatus('準備完了');` とハードコーディングされている計3箇所を、以下のロジックに置き換えます。
  - `appState.settings.saveMode` を判定
  - true（自動保存）なら `t('status_ready_auto')` を渡す
  - false（手動保存）なら `t('status_ready_manual')` を渡す

### [MODIFY] `src/dist/js/ui/tabs.js`
- 既存の `updateStatus('保存準備完了', ...)` というベタ書き箇所も、同様に保存モードを判定して定数化された翻訳キーを呼び出す形式へ修正します。

## 4. 検証プラン
1. アプリを起動し、起動直後のステータスバーが現在の保存モードに応じた正しい文言（`i18n.js`から取得されたもの）になっていることを確認する。
2. 手動保存モードに切り替えた際、ステータスバーの文言が正しく定数から取得され表示されることを確認する。
