# 文字数カウント設定機能実装の変更内容（Walkthrough）

設定画面から文字数カウント方法を選択・永続化できる機能の実装が完了しました。

## 変更内容

### バックエンド
#### 1. [main.rs](file:///c:/work/NoCapEdit/src/main.rs)
- `AppSettings` および `SettingsResponse` 構造体に `char_count_mode` フィールド（デフォルト値 `"with_newline"`）を追加しました。
- `get_settings` および `save_settings` Tauriコマンドを変更し、設定ファイルの読み書き処理を新項目に対応させました。

### フロントエンド
#### 2. [index.html](file:///c:/work/NoCapEdit/src/dist/index.html)
- 設定ダイアログ（モーダル）内に、文字数カウント方法を選択するためのドロップダウンリスト（`#charCountModeSelectModal`）を追加しました。

#### 3. [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
- `appState` に `charCountMode` 状態を追加し、初期起動時の値読み込みおよびモーダルへの反映（UI同期）を実装しました。
- 設定変更イベントを監視し、選択値が変更されたらバックエンドの `save_settings` コマンドで即座に設定を永続化するようにしました。
- `updateEditorMetrics` の文字数カウントロジックを、設定値（`with_newline` または `no_newline`）に応じて分岐するよう修正しました。これにより、設定項目がリアルタイムに反映されます。
