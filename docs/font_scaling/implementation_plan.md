# フォント拡大縮小機能の実装計画

エディタの利便性向上のため、フォントの拡大縮小機能を追加します。
拡大縮小のトリガーとして以下の2つの方法をサポートします。
1. `Ctrl` + マウスホイール回転
2. `Ctrl` + `+`（拡大） / `-`（縮小）

また、変更されたフォントサイズはアプリケーションの設定（`config.json`）に保存し、再起動後も維持されるようにします。

## ユーザーレビューが必要な項目

> [!NOTE]
> - フォントサイズの最小値は `8px`、最大値は `72px`、変更ステップは `1px` とします。
> - ショートカットキー `Ctrl` + `+` については、日本語キーボードや英語キーボードの差異を考慮し、`Shift` キーの有無に関わらず `+` キー（または `;` や `=` キー）の押下で拡大動作が動作するように調整します。

## 提案する変更内容

### バックエンド (Rust)

#### [MODIFY] [main.rs](file:///c:/work/NoCapEdit/src/main.rs)
- `AppSettings` 構造体に `font_size` フィールド（デフォルト13）を追加します。
- `SettingsResponse` に `font_size` を追加し、フロントエンドに返却するようにします。
- `save_settings` コマンドの引数に `font_size` を追加し、設定ファイルに保存できるようにします。

### フロントエンド (HTML/JS/CSS)

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
- `appState` に `fontSize` を追加し、初期化時に `get_settings` から取得した値をエディタに適用します。
- テーマのトグル時や初期設定保存時の `save_settings` 呼び出しに `fontSize` パラメータを追加します。
- 以下のズーム用ヘルパー関数を追加します：
  - `zoomIn()`: フォントサイズを1px増やし、適用・保存する。
  - `zoomOut()`: フォントサイズを1px減らし、適用・保存する。
  - `applyFontSize()`: エディタ要素のスタイルにフォントサイズを反映し、設定保存処理をトリガーする。
  - `saveFontSizeDelay()`: 頻繁なディスク書き込みを防ぐため、1秒のデバウンスを挟んで `save_settings` を呼び出す。
- `setupUIEventListeners` にて、以下のグローバルイベントリスナーを登録します：
  - `wheel` イベント: `ctrlKey` が押されている場合、スクロール方向に応じて `zoomIn()` / `zoomOut()` を呼び出し、デフォルトのブラウザズームを防止します。
  - `keydown` イベント: `ctrlKey` が押されている場合、`+` や `-` に関連するキー入力を検知して `zoomIn()` / `zoomOut()` を呼び出し、デフォルトのキー入力を防止します。

#### [MODIFY] [style.css](file:///c:/work/NoCapEdit/src/dist/style.css)
- エディタの初期フォントサイズ（`.editor`）が現在 `13px` に設定されていることを確認し、JS側のデフォルト値と整合性を保ちます（変更なし、または微調整）。

## 検証計画

### 手動検証
1. **初期状態確認**: アプリ起動時にエディタのフォントサイズがデフォルト（13px）であることを確認します。
2. **Ctrl + ホイール操作**:
   - `Ctrl` キーを押しながらマウスホイールを上に回転させるとフォントが大きくなることを確認します。
   - 下に回転させるとフォントが小さくなることを確認します。
   - `8px` 未満、および `72px` 超過にはならないことを確認します。
3. **Ctrl + +/- ショートカット**:
   - `Ctrl` + `+`（日本語キーボードでは `Ctrl` + `;` または `Ctrl` + `Shift` + `;`）でフォントが大きくなることを確認します。
   - `Ctrl` + `-`（日本語キーボードでは `Ctrl` + `-`）でフォントが小さくなることを確認します。
4. **設定の永続化**:
   - フォントサイズを変更した状態でアプリを再起動し、変更後のフォントサイズが維持されていることを確認します。
   - `config.json` を確認し、`font_size` が正しく保存されていることを確認します。
