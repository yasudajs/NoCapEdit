# ワークスルー - フォント拡大縮小機能の実装

フォントの拡大縮小機能の実装と、設定の永続化が完了しました。

## 変更内容

### 1. バックエンド (Rust)
- [main.rs](file:///c:/work/NoCapEdit/src/main.rs)
  - `AppSettings` 構造体に `font_size` フィールド（デフォルト `13`）を追加。
  - `SettingsResponse` に `font_size` を追加し、フロントエンドに返却するように修正。
  - `save_settings` コマンドで `font_size` を引数として受け取り、設定ファイル (`config.json`) に保存できるように修正。

### 2. フロントエンド (JS)
- [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
  - アプリケーション起動時に設定から `font_size` を読み込み、エディタ要素 (`elements.editor`) に適用する機能を追加。
  - `zoomIn()` / `zoomOut()` 関数を実装（最小 8px、最大 72px、ステップ 1px）。
  - マウスホイールイベントリスナーを追加。`Ctrl` キー押下時のホイールスクロールを検知してフォントサイズを変更し、ブラウザ本来のズームを抑制。
  - キーボードショートカットイベントリスナーを追加。`Ctrl` + `+`/`-` （日本語配列の `;`/`-` やテンキーの `+`/`-` にも対応）でのフォントサイズ変更を処理。
  - フォントサイズ変更時に、1秒のデバウンス（遅延）を置いて `save_settings` を呼び出し、頻繁な書き込みを防ぎつつ自動保存する機能を追加。
  - 設定トグルや初期起動時保存の際にもフォントサイズ設定を保持したまま `save_settings` を呼び出すように同期。

## 検証結果

- **ビルド確認**: `cargo check` および `cargo build` がエラーなしで成功することを確認しました。
- **手動確認依頼項目**:
  1. `Ctrl` + マウスホイールを上下に回転させ、フォントサイズがスムーズに拡大・縮小すること。
  2. `Ctrl` + `+` (または `;`) および `Ctrl` + `-` キーを押下し、フォントサイズが拡大・縮小すること。
  3. フォントサイズ変更後、アプリを一度終了して再起動した際、フォントサイズが変更後の状態のまま維持されること。
