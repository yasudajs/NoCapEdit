# ウォークスルー: ステータスバーのファイル状態表示バグ修正

ステータスバーの表示不具合に対応するための実装とビルド検証が完了しました。

## 変更内容

### フロントエンド修正

#### [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
1. **`saveTabIfDirty(tab)`** のシグネチャ変更
   - 実際に保存が実行された場合は `true`、スキップされた場合は `false` を返すよう修正しました。
2. **`triggerManualSave()`** および **`autoSave()`** の修正
   - `saveTabIfDirty` の戻り値が `false`（保存がスキップされた）の場合、ステータスを「〜を作成」に更新せず、`updateTabStatus(tab)` を呼んで現在のタブ状態に即したステータス表示（新規空ファイルなら「保存準備完了」等）に戻すようにしました。
3. **`closeTab(tabId)`** の修正
   - アクティブなタブを閉じた後のタブ切り替え処理で、直接 `appState.currentTab` を代入するのではなく、`await switchTab(appState.tabs[0].id)` を呼び出すように変更しました。これにより、ステータスバーの更新処理やエディタの各種状態復元（カーソル等）が一貫して実行されるようになりました。
