# ウォークスルー：起動時のウィンドウ表示ガタつき修正 (Prevent Startup Flicker)

起動時にOSのウィンドウ枠が表示された後、WebViewのレンダリングや初期化非同期処理（IPC通信など）が完了するまで画面の描画がガタついて見える問題を解消するため、起動時はウィンドウを非表示にし、初期表示用データの準備が完全に整ったタイミングで画面を一気に表示する仕様に変更しました。

## 変更内容

### 1. Rustバックエンド (`src/main.rs` / `Cargo.toml` / `tauri.conf.json`)
- **ウィンドウの初期非表示化**: `WindowBuilder` でメインウィンドウを構築する際、`.visible(false)` を追加し、起動直後のウィンドウがバックグラウンドに置かれるようにしました。
- **Tauriフィーチャーの有効化**: JavaScriptからウィンドウを表示するために必要な `"window-show"` フィーチャーを `Cargo.toml` の `tauri` に追加しました。
- **セキュリティ許可 (Allowlist) の追加**: `tauri.conf.json` の `allowlist.window` に `"show": true` を追加し、フロントエンドから `appWindow.show()` を安全に呼び出せるようにしました。

### 2. フロントエンド (`src/dist/main.js`)
- **初期化完了後の表示制御 (finallyによるフェイルセーフ)**: `init()` 関数の終了部分に `finally` ブロックを導入し、正常に初期化が完了した時はもちろん、初期化中に何らかのエラー（例外）が発生した場合であっても確実に `appWindow.show()` が呼び出され、画面が表示されるようにリファクタリングしました。

### 3. 仕様書 (`docs/spec.md`)
- 「3.3 ウィンドウ表示制御（起動時ガタつき防止）」セクションを追加し、今回の起動時非表示および初期化完了後の表示制御について仕様化しました。

## 検証結果

### 1. ビルド検証
- `cargo build` を実行し、警告やエラーがなく正常にビルドが成功することを確認しました。

### 2. 手動検証
- 修正版の配布用パッケージ（NSIS版）をビルドしてインストールし、アプリアイコンのクリック起動時に、ウィンドウ枠のみが表示されてから一瞬遅れて中身が表示されるガタつき（チラつき）が解消され、完全に描画が整った状態で綺麗に一発で表示されることを確認しました。
