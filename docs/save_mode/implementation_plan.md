# 保存モード切り替え機能の実装計画

## 概要
設定画面にて、現在の強制保存仕様（自動保存）と、「無保存モード（手動保存）」を切り替えられる機能を追加します。
無保存モードでは、自動保存は行われず、閉じる際の確認も出さずに内容は破棄されます。また、手動で保存した場合は一時的にファイルへ出力（スナップショット）しますが、タブ自体は無保存モードを維持します。

## 実装内容（提案される変更）

### バックエンド (Rust)
#### [MODIFY] `src/main.rs`
- `AppSettings` 構造体に `save_mode: String` フィールドを追加。
- デフォルト値を `"auto"` とするための関数 `default_save_mode()` を追加し、`#[serde(default = "default_save_mode")]` を設定。

### フロントエンド (HTML/CSS)
#### [MODIFY] `src/dist/index.html`
- `<div id="settingsDialog">` の設定項目内に「保存モード」のセレクトボックス (`#saveModeSelectModal`) を追加。
  - `<option value="auto">自動保存（入力時に自動保存）</option>`
  - `<option value="manual">無保存（手動保存のみ）</option>`

### フロントエンド (JavaScript)
#### [MODIFY] `src/dist/main.js`
- **設定の管理**
  - アプリの初期化時および設定画面表示時に `appState.settings.saveMode` を読み込み、UIに反映。
  - 設定の保存 (`saveSettings`) 時に `saveMode` の値をバックエンドに送信。
  - 設定変更直後、開いている全タブに対して即座にモード変更を適用（例えば、`auto` → `manual` への切り替えで自動保存タイマーをキャンセルなど）。
- **タブ作成処理 (`createNewTab`)**
  - 現在のモードが `'manual'`（無保存モード）の場合：
    - バックエンド側でのファイル作成（`invoke('create_text_file', ...)` 等）をスキップ。
    - タブの `fileName`（または表示用のタイトル）を `[yyyy/mm/dd hh:mm:ss]` に設定。
    - 内部的な状態としてファイルパスは空または専用のダミー値を持つ。
- **ステータス表示 (`updateStatus` などの拡張)**
  - `saveMode === 'manual'` の場合、ステータスバー左下に常に `[無保存モード]` と表示する。
- **自動保存と閉じる際の処理 (`autoSave`, `persistTabWithRecovery`, `closeTab`)**
  - エディタ入力時の自動保存タイマー起動を `saveMode === 'auto'` の場合のみに制限。
  - `saveMode === 'manual'` の場合、タブを閉じる際やウィンドウを閉じる際の `persistTabWithRecovery` などのリカバリ保存や確認をバイパスし、即座に破棄して閉じる。
- **手動保存処理 (`triggerManualSave`)**
  - `saveMode === 'manual'` の場合、`Ctrl + S` が押された際はダイアログを表示せず、直接ホームフォルダに保存（`invoke('save_text_file')`）する。
  - この時のファイル名として、タブの表示名が `[yyyy/mm/dd hh:mm:ss]` であればそこから時間を抽出し、`yyyymmdd_hhmmss.nctx` の形式で保存する。
  - 保存完了後も、タブは通常のファイルを持つタブには昇格させず、無保存タブとしての状態を維持する。
- **モード切り替え時の処理（`saveSettings`）**
  - **`manual` → `auto`**: 実体ファイルが存在しない（`filePath` が空の）タブに対して、強制的に実体ファイル（空ファイル）を作成・割り当てし、正式な `fileName` と `filePath` を設定して自動保存を再開する。
  - **`auto` → `manual`**: 中身が未入力のままのタブ（空ファイルが紐付いているタブ）について、設定変更時にその場で実体ファイルを削除し、タブ情報を無保存モードの初期状態（ファイル未紐付け、タイムスタンプ名の表示）にリセットする。
- **空ファイルの確実な破棄処理（`persistTabWithRecovery`）**
  - タブを閉じる際に、タブの状態が「無保存モード」であっても、もし実体ファイルが存在して中身が空であれば、削除処理を最優先で実行するように順序を変更する。

## 確認（Verification Plan）
### 自動テスト
- 現在のプロジェクトに自動テストの記述がないため、手動テストをメインとします。ビルドが通るか (`cargo build`) 確認します。

### 手動テスト
1. **設定の切り替え**: 設定画面で保存モードを切り替え、正しく `config.json` に保存されるか確認する。
2. **無保存モードの挙動確認**:
   - 新規タブを作成した際、ファイルが作成されず、タブ名が `[yyyy/mm/dd hh:mm:ss]` になっているか。
   - ステータスバーに `[無保存モード]` と表示されているか。
   - 文字を入力しても自動保存されないか。
   - 未保存のままタブを閉じた際に、警告なしで閉じられるか。
3. **無保存モードでの手動保存**:
   - `Ctrl + S` を押した際、ホームフォルダに `yyyymmdd_hhmmss.nctx` がダイアログなしで作成されるか。
   - 保存後もタブが「無保存モード」のまま維持されているか（追記して閉じた場合、追記分が保存されないか）。
4. **途中切り替えの挙動**:
   - 自動保存モードで開いていたタブがある状態で無保存モードに切り替えた際、以降の自動保存が停止するか。
   - その逆の際、正常に自動保存が再開されるか。
