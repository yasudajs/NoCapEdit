## 提案される変更点

### バックエンド (Rust)

#### [MODIFY] [main.rs](file:///c:/work/NoCapEdit/src/main.rs)
以下の2つの Tauri コマンドを定義し、ビルダーの `invoke_handler` に登録します。

1. **`delete_file_or_dir_permanently`**
   * 引数: `file_path: String`
   * セキュリティチェック: 対象パスが `home_folder` 内にあるか (`starts_with`)。
   * 削除処理: 対象がファイルなら `fs::remove_file`、ディレクトリなら `fs::remove_dir_all` を実行。

2. **`move_file_or_dir`**
   * 引数: `source_path: String`, `target_parent_path: String`
   * セキュリティチェック: 移動元・移動先の双方が `home_folder` 内にあるか。
   * 循環移動防止: 移動先フォルダが移動元フォルダの配下（自身または子孫）でないかチェック。
   * 同名衝突防止: 移動先に同名のファイルまたはフォルダが既に存在しないかチェック。
   * 移動処理: `fs::rename(&canon_src, &dest_path)` を実行。

### フロントエンド (JS/CSS)

#### [MODIFY] [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
1. **選択状態の管理**:
   * モジュールローカルに `selectedPath` と `selectedElement` を保持。
   * `renderFileTree` でツリーアイテム（`.tree-item`）を描画する際、クリックイベントにて `selectedPath` / `selectedElement` を更新し、`.selected` クラスを着脱する。
   * ツリー背景クリック時、およびエディタクリックなどのフォーカスアウトイベント時に、`.selected` クラスを取り除き、代わりに `.selected-inactive` クラスを付与する。
2. **キー監視**:
   * グローバルキーダウンリスナーを登録（ただし、エディタの入力中やダイアログ入力欄のアクティブ時は除外）。
   * `Delete` 押下時: 選択中のパスがあれば通常の削除（ゴミ箱移動）を実行。
   * `Shift + Delete` 押下時: 選択中のパスがあれば完全削除 `deleteItemPermanentlyInTree` を実行。
3. **完全削除の処理 (`deleteItemPermanentlyInTree`)**:
   * `window.__TAURI__.dialog.ask` で「本当に完全に削除しますか？この操作は取り消せません。」を表示。
   * 承認された場合、Rustの `delete_file_or_dir_permanently` を呼び出し、成功したらツリーを部分再描画して、開いているタブがあれば閉じる。
4. **ドラッグ＆ドロップの実装**:
   * ツリーアイテムを `draggable="true"` にする。
   * イベントハンドラを設定:
     * `dragstart`: `e.dataTransfer.setData('text/plain', filePath)`。ドラッグ元のパスやフォルダ属性を保持。
     * `dragover`: 自分自身、自分の子フォルダ、または現在の親フォルダへのドロップでなければ `e.preventDefault()` で許可。
     * `dragenter` / `dragleave`: ターゲットフォルダに `.drag-hover` クラスを付与・削除。
      * `drop`: ドロップ先がファイルならその親フォルダ、フォルダならそのフォルダ自体を宛先とし、Rustの `move_file_or_dir` を呼び出す。
      * 不正な移動（自分自身やその子フォルダへの移動、同一階層への移動など）やAPIエラー時、モーダルダイアログは出さず、ステータスバー（`statusText`）に赤文字でエラーメッセージを一時表示（3秒後に自動クリア）する。
      * 成功時、移動元フォルダおよび移動先フォルダの再読み込みを実行。

#### [MODIFY] [style.css](file:///c:/work/NoCapEdit/src/dist/style.css)
* `.tree-item.selected` : アクティブな選択ハイライト（例：メインカラー系の背景）。
* `.tree-item.selected-inactive` : 非アクティブな選択ハイライト（例：薄いグレー系の背景）。
* `.tree-item.drag-hover` : ドラッグホバー時の簡易ハイライト（例：わずかに明るい背景または破線のアウトライン）。

## 検証計画

### 自動テスト
* なし。

### 手動検証
1. **選択ハイライトと非アクティブ選択**:
   * ツリーのファイルやフォルダをクリックした際、ハイライトされること。
   * エディタ側の文字入力部分をクリックした際、ツリーのハイライトが薄いグレーに変わること。
   * サイドバー外（エディタなど）をクリックした状態で `Delete` を押しても削除が発動しないこと。
2. **完全削除の確認**:
   * 項目を選択し `Shift + Delete` を押した際、「完全に削除しますか？」の警告ダイアログが表示されること。
   * ダイアログで「はい」を選択した際、ファイル/フォルダが消滅し、OSのゴミ箱にも入っていないこと。
3. **ドラッグ＆ドロップ移動**:
   * ファイルを別のフォルダにドラッグ＆ドロップし、そのフォルダの下に移動すること。
   * ファイルを別の「ファイル」の上にドロップした際、そのファイルの親フォルダ内に移動すること。
   * フォルダを自分自身、または自分自身の子フォルダ内に移動させようとした際、モーダルダイアログは表示されず、ステータスバーに赤文字でエラーメッセージが一時表示（数秒間）され、移動できないこと。
