# v0.1系 ファイル構造共通化（モジュール分割 ＆ i18n導入）実装計画

v0.1系（`master` ブランチ）と v0.2系（`v0.2` ブランチ）の並行運用におけるメンテナンス性を向上させるため、v0.1系のフロントエンド JavaScript 構造を v0.2系と同等のモジュール分割構造にリファクタリングし、多言語対応の仕組み（`i18n.js`）を導入します。

これにより、両ブランチ間でのバグ修正や共通機能の同期（チェリーピック等）を容易にします。

---

## ユーザーレビューが必要な項目

> [!IMPORTANT]
> **v0.1系の機能範囲の維持について**
> 今回のリファクタリングでは、**v0.1系に実装されている機能のみ**を対象とします。v0.2系で追加された「サイドバー（ファイルツリー）」やその関連操作（ドラッグ＆ドロップ、フォルダ削除の警告等）は移植対象外とし、純粋にv0.1系の「エディタ機能」「タブ機能」「設定画面（右ドック）」のロジックをモジュール分割します。
→　はい、それでOKです。

## 提案する変更内容

### フロントエンド（モジュール分割とi18nの導入）

`src/dist/main.js`（約1550行）の巨大な単一ファイルを廃止し、以下のように役割ごとにモジュール分割します。

#### [NEW] [i18n.js](file:///c:/work/NoCapEdit/src/dist/i18n.js)
* 言語辞書オブジェクト `DICT` と、翻訳関数 `window.t` を定義します。
* v0.1系のダイアログ確認メッセージやエラー文をここに定義します。

#### [NEW] [js/main.js](file:///c:/work/NoCapEdit/src/dist/js/main.js)
* アプリケーションの最初のエントリポイントです。
* 各種DOM要素の取得、グローバルイベントリスナーの登録、初期化処理（テーマ・フォント設定のロードなど）を行います。

#### [NEW] [js/state.js](file:///c:/work/NoCapEdit/src/dist/js/state.js)
* アプリケーション全体のグローバル状態（`appState`）の定義と初期化を行います。

#### [NEW] [js/ui/editor.js](file:///c:/work/NoCapEdit/src/dist/js/ui/editor.js)
* エディタ（`textarea`）に対する操作、文字数カウント、フォントサイズ/行間の調整処理を定義します。

#### [NEW] [js/ui/tabs.js](file:///c:/work/NoCapEdit/src/dist/js/ui/tabs.js)
* タブの追加・切り替え・削除・レンダリングのロジック、およびカーソル位置の復元ロジックを管理します。

#### [NEW] [js/ui/settings.js](file:///c:/work/NoCapEdit/src/dist/js/ui/settings.js)
* 設定画面（右ドック）の開閉、テーマ切り替え、フォントファミリーの遅延ロードなどのUI制御を管理します。

#### [NEW] [js/core/fileSystem.js](file:///c:/work/NoCapEdit/src/dist/js/core/fileSystem.js)
* ファイルの遅延作成、自動保存、安全保存方式（一時ファイルを使った置換保存）、保存失敗時のリカバリなどのファイルI/Oに関連するロジックを管理します。

#### [NEW] [js/core/tauri.js](file:///c:/work/NoCapEdit/src/dist/js/core/tauri.js)
* Tauri API（`invoke`、ファイルダイアログ、シェルオープンなど）の呼び出しをラッピングし、JSモジュールから利用しやすくします。

#### [NEW] [js/utils/helpers.js](file:///c:/work/NoCapEdit/src/dist/js/utils/helpers.js)
* デバウンス関数や時間フォーマット変換などの汎用的なユーティリティ関数を定義します。

#### [DELETE] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
* 古い単一のJSファイルを削除します。

#### [MODIFY] [index.html](file:///c:/work/NoCapEdit/src/dist/index.html)
* 古い `<script src="main.js"></script>` を削除し、以下のようにモジュールとしてスクリプトを読み込みます。
  ```html
  <script src="i18n.js"></script>
  <script type="module" src="js/main.js"></script>
  ```

## 検証計画

### 自動テスト
* `cargo test` を実行し、Rustバックエンドのユニットテストが正常に通ることを確認します。

### 手動検証
リファクタリングの前後で、以下のv0.1系のコア機能がすべて不具合なく動作することを確認します。

1. **起動とファイル自動生成（遅延作成）**
   * 起動時に新規タブ（未保存1）が立ち上がり、入力を開始して3秒後にホームフォルダ配下に `yyyymmdd_hhmmss.nctx` が自動生成されること。
2. **タブ管理**
   * タブの追加（プラスボタン）、タブ切り替え、タブ削除（クリーンアップ含む）が正常に機能すること。
   * タブ切り替え時にカーソル位置とスクロール位置が正常に復元されること。
3. **設定（右ドック）とテーマ**
   * 設定画面が開き、テーマ（Dark / Soft Dark / Light）の変更が即座に反映されること。
   * フォントサイズのズーム（`Ctrl` + マウスホイール）、行間調整が機能し、設定ファイルに保存されること。
   * フォントファミリーのセレクトボックスをクリックした際にシステムフォントが遅延ロードされ、適用できること。
4. **自動保存 & 安全保存**
   * 入力停止時、タブ非アクティブ時、アプリ終了時に自動保存され、ファイル内容が安全にディスクに書き込まれること。
