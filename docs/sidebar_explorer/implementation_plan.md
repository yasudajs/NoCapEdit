# 実装計画書 - Phase 5.3: ツリー更新時のスクロール・選択状態の完全保持

本フェーズでは、ファイルツリーの更新（外部監視、自動保存、手動リロード）が発生した際に、スクロール位置、フォーカス状態、およびフォルダの展開状態を完全に維持するため、ツリー全体を再生成（`innerHTML = ''`）するのではなく、変更があった箇所のみをインクリメンタルに同期する「差分更新」方式を導入します。
また、ライトモードのツリーにおけるアクティブな選択ハイライト色を、より優しい薄青色（`#CCE8FF`）に変更します。

## ユーザー確認事項

> [!IMPORTANT]
> - ファイルツリーの再描画時の挙動が「丸ごと破棄・再生成」から「DOMを再利用した差分更新」に変わります。これにより、既存のツリーアイテムが持つイベントリスナーやフォーカスが自然に維持されるようになります。
> - ライトモードでツリー項目にフォーカスがあるとき（アクティブ選択状態）の背景色が、濃い青から薄い青（`#CCE8FF`）に変わります。これに伴い、テキストカラーも白文字から通常の濃いグレー（`var(--text-primary)`）に変更し、視認性を確保します。

### フロントエンド UI

#### [MODIFY] [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
- `renderFileTree` 関数を、差分更新（既存DOMスキャン ➔ 不要DOM削除 ➔ 新規DOM作成・既存DOM再利用および順序整列）をおこなうアルゴリズムに変更します。
- コードの可読性と再利用性を高めるため、個々のツリー項目（`.tree-item`）を生成する処理をヘルパー関数 `createTreeItemElement(file)` に分割・整理します。
- 差分更新の適用時、リロード対象のコンテナに古いエラー表示や空表示がある場合、およびパスが変わる完全リフレッシュ時（ホームフォルダ変更など）には、従来通り `innerHTML = ''` による完全リビルド（フルリロード）を行うパスを残します。

#### [MODIFY] [style.css](file:///c:/work/NoCapEdit/src/dist/style.css)
- ライトテーマ（`body.light-theme`）における、選択中のツリー項目（`.tree-item.selected`）のスタイルを追加し、背景色を `#CCE8FF`、文字色を `var(--text-primary)`（または適切な濃い文字色）に設定します。

## 検証計画

### 自動テスト
現時点でフロントエンド UI 向けの自動統合テストはないため、コンパイル（ビルド）および手動テストによって動作を確認します。
- `cargo tauri dev` によるビルドと起動テスト

### 手動検証
以下の項目を動作確認し、`docs/wip/sidebar_explorer/task.md` にチェックリストとして記録します。

1. **スクロール位置の維持**:
   - ツリーを長く下にスクロールさせた状態で、外部のファイルをエディタ内で編集・自動保存した際、または外部でファイルを作成した際に、スクロール位置（`scrollTop`）が一切ブレないことを確認する。
2. **フォーカスと選択状態の維持**:
   - ツリー内でいずれかのファイル（またはフォルダ）を選択（フォーカス）している状態で、手動リロード（`F5` やファイル監視イベント）をトリガーした際、フォーカスおよび青/グレーの選択表示が外れずに維持されることを確認する。
3. **展開状態の維持**:
   - ネストしたフォルダを展開した状態で、自動保存や外部ファイル変更による更新が走った際、展開（📂）されていたフォルダが勝手に閉じないことを確認する。
4. **新規作成と名前変更**:
   - `Ctrl+N` / `Ctrl+D` での新規作成や, `F2` でのインライン名前変更が差分更新と競合せず、正常に実行できることを確認する。
5. **ライトモードの選択カラー**:
   - ライトモードに切り替え、ツリーで項目を選択した際、ハイライトの背景色が優しい青（`#CCE8FF`）になり、文字がはっきり読めることを確認する。
