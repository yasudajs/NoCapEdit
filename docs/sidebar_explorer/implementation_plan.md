# サイドバー（ファイルエクスプローラ）でのファイル操作（作成・名前変更・通常削除）の実装計画

サイドバーのファイルツリーから直接ファイル・フォルダの作成、名前変更、通常削除（ゴミ箱移動）を行えるようにします。また、操作がエディタ（タブ）と連動する仕組みを構築します。

## ユーザー確認事項
* 今回、ファイル通常削除時にOSのゴミ箱へ移動させるため、Rustの `trash` クレート（v3.0）を `Cargo.toml` に追加します。

## 提案する変更

---

### バックエンド (Rust)

#### [MODIFY] [Cargo.toml](file:///c:/work/NoCapEdit/Cargo.toml)
* `trash` クレートを追加し、安全なゴミ箱移動機能を実装できるようにします。
```toml
trash = "3.0"
```

#### [MODIFY] [main.rs](file:///c:/work/NoCapEdit/src/main.rs)
* 以下のTauriコマンドを新規追加・登録します：
  1. `create_file_or_dir(parent_path: String, name: String, is_dir: bool) -> Result<String, String>`
     * 指定された親ディレクトリ配下に、ファイルまたはフォルダを新規作成します。
     * ディスク上に既に同名ファイルが存在する場合、ファイルであれば `[stem]_[連番].[ext]`、フォルダであれば `[名前]_[連番]` のように連番を自動付与して衝突を回避します（例: `名称未設定_01.nctx`）。
     * 作成完了したファイルの絶対パスを返します。
  2. `rename_file_or_dir(old_path: String, new_name: String) -> Result<String, String>`
     * ファイル・フォルダの名前を変更します。すでに変更先が存在する場合はエラーを返します（名前変更時は自動連番にせずエラーにします）。
     * 変更後の絶対パスを返します。
  3. `trash_file_or_dir(file_path: String) -> Result<(), String>`
     * 指定のパスをOSのゴミ箱に移動します。
* すべてのコマンドで、セキュリティチェックとしてパスが設定された `home_folder` の配下にあることを検証します。

#### [MODIFY] [index.html](file:///c:/work/NoCapEdit/src/dist/index.html)
* サイドバー用のカスタムコンテキストメニューDOM（`#context-menu`）を追加します。
  * メニュー項目：`新規ファイル作成`, `新規フォルダ作成`, `名前を変更`, `削除`

#### [MODIFY] [style.css](file:///c:/work/NoCapEdit/src/dist/style.css)
* カスタムコンテキストメニューのスタイル（`.context-menu` 等）を追加します。
  * アプリのアクティブなテーマ（Dark/Soft Dark/Light）に応じた背景色、文字色、ボーダーを設定し、Obsidian風のフラットでモダンなUIデザインを採用します。
* インライン入力用のインプット（`.tree-input`）のスタイルを追加します。ツリーアイテム内でフォントや位置が崩れないように調整します。

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
* **コンテキストメニュー制御**:
  * 右クリックイベント（`contextmenu`）をツリー要素で検出し、表示座標に合わせてメニューを表示します。
  * 右クリックされた対象（ファイル、フォルダ、または背景）を判別し、表示する項目を制御します：
    * **ファイル**: `名前を変更`, `削除`
    * **フォルダ**: `新規ファイル作成`, `新規フォルダ作成`, `名前を変更`, `削除`
    * **背景**: `新規ファイル作成`, `新規フォルダ作成`
  * メニュー外クリックまたは `Esc` キーでメニューを非表示にします。
* **インライン編集（作成・名前変更）**:
  * `新規ファイル作成`, `新規フォルダ作成`, `名前を変更` を選択した際、対象の要素をテキスト入力欄（`<input>`）に置き換えます。
  * 新規作成の場合、親フォルダの子要素の末尾にダミーのアイテムを追加して入力欄を表示します（自動的に親フォルダを展開）。
  * テキスト入力欄にフォーカスを当て、初期テキストを選択状態にします（名前変更時は拡張子を除いたベース名のみを選択します）。
  * `Enter` または `blur` で確定APIを呼び出し、`Esc` でキャンセルします。
  * 入力値が空の場合、新規作成時はデフォルト値（`名称未設定.nctx` / `新しいフォルダ`）を採用し、名前変更時はキャンセル扱いとします。
  * ファイル名に許可された拡張子（`.nctx`, `.txt`, `.md`, `.json`, `.csv`）が含まれていない場合、自動的に末尾に `.nctx` を付与します。
  * ファイル新規作成が成功したら、自動的にそのファイルをエディタの新規タブとして開きます。
* **削除とエディタ連携**:
  * `削除` を選択した際、Tauriの `dialog` APIを使って「本当に削除しますか？」の確認ダイアログを表示します。
  * 削除に成功したらツリーを更新し、削除されたファイルがエディタで開かれている場合はそのタブを自動的に（保存確認を挟まずに）閉じます。
* **名前変更とエディタ連携**:
  * 名前変更に成功したらツリーを更新し、そのファイルがエディタで開かれている場合は、タブオブジェクトのパス情報を新しいパスに更新し、タブの表示名も新しい名前に連動して変更します。

## 検証計画

### 1. 手動テスト
1. **コンテキストメニューの表示テスト**:
   * ファイル、フォルダ、背景のそれぞれを右クリックし、適切なメニュー項目が表示されることを確認します。
   * メニュー外クリックや `Esc` でメニューが正しく閉じることを確認します。
   * テーマ（Dark / Soft Dark / Light）を切り替えて、メニューのデザインが崩れず追従することを確認します。
2. **新規作成（インライン編集）**:
   * 空の入力で新規ファイルを作成し、`名称未設定.nctx` が作成され、自動的にエディタのタブとして開かれることを確認します。
   * 再度空の入力で新規ファイルを作成し、衝突回避によって `名称未設定_01.nctx` などが作成されることを確認します。
   * 拡張子なし（例: `test`）で入力した際、`test.nctx` として作成されることを確認します。
   * 許可されない拡張子（例: `test.png`）で入力した際、自動で `test.png.nctx` のように補完されるか、またはエラーになることを確認します。
   * 新規フォルダの作成でも同様に動作（デフォルト名: `新しいフォルダ`、衝突時の連番）することを確認します。
3. **名前変更（インライン編集）**:
   * ファイル・フォルダの名前を正常に変更できることを確認します。
   * すでに存在する名前へ変更しようとした際、適切なエラーが発生し変更されないことを確認します。
   * エディタで開いているファイルの名前を変更した際、タブの表示名が追従し、内部の保存先パス情報も書き換わることを確認します。
4. **削除動作と連動**:
   * 削除時に確認ダイアログが表示されることを確認します。
   * 削除を実行した際、ファイルが実際にOSのゴミ箱へ移動していることを確認します。
   * エディタで開いているファイルを削除した際、エディタのタブが自動で閉じられることを確認します。
