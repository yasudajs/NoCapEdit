# Phase 2: ファイルツリーの表示とエディタ連携 実装計画

## 目標
サイドバー（エクスプローラ領域）に、基準ディレクトリ内のファイルとフォルダをツリー形式（階層型）で表示し、フォルダの展開・折りたたみ、およびファイルのクリックによるエディタ連携（新しいタブで開く）を実現します。

## 仕様の決定事項（前回のディスカッションより）
1. **表示階層と展開機能**: 初期表示は基準ディレクトリ直下とし、フォルダをクリックした際に動的に中身を取得して展開表示（アコーディオン）できるようにする。
2. **ファイルのフィルタリング**: エディタで直接開けるテキスト系ファイルのみをツリーに表示する（対象拡張子: `.txt`, `.md`, `.nctx`, `.json`, `.csv`）。
3. **基準ディレクトリの制限**: ホームフォルダ直下の設定ディレクトリ（`NoCapEdit_Doc`等）を操作対象とし、それより上の階層へは移動・参照できないようにする。

## 実装案（アーキテクチャ変更）

### 1. バックエンド (Rust - `src/main.rs`)
- **ファイル情報構造体の定義**: 
  ファイル名、絶対パス、ディレクトリかどうか(`is_dir`)などを保持する `FileInfo` 構造体を定義し、`Serialize` を実装します。
- **新規Tauriコマンドの追加**: 
  `read_directory(path: Option<String>) -> Result<Vec<FileInfo>, String>`
  - 引数 `path` が `None` の場合は `AppSettings.home_folder` を対象とします。
  - 指定されたパスが `AppSettings.home_folder` の配下にあるかどうかを検証し、上位階層へのアクセス（ディレクトリトラバーサル等）をブロックします。
  - ディレクトリ内のエントリを読み込み、フォルダ、および許可された拡張子（`.txt`, `.md`, `.nctx`, `.json`, `.csv`）のファイルのみを抽出します。
  - 抽出したリストを「フォルダが上、ファイルが下（それぞれ名前順）」になるようソートして返却します。

### 2. フロントエンドロジック (JavaScript - `main.js`)
- **API連携**:
  `invoke('read_directory')` を呼び出し、ファイル一覧を取得する非同期関数 `loadDirectory(path, parentElement)` を実装します。
- **DOMレンダリング（動的生成）**:
  取得した `FileInfo` リストをもとに、対象の要素内にHTML要素（`div.tree-item` など）を生成します。
  - フォルダの場合は「📁 フォルダ名」、ファイルの場合は「📄 ファイル名」のようなアイコンを付与します。
- **クリックイベントのバインド**:
  - **ファイルをクリックした場合**：ファイルパスを引数にして既存の読み込み処理を呼び出し、エディタの新しいタブ（または既存のタブ）に内容を表示します。
  - **フォルダをクリックした場合**：すでに展開済みの場合は子要素を非表示（折りたたみ）にします。未展開の場合は `loadDirectory` を呼び出して子要素を取得・レンダリングし、展開します。

### 3. フロントエンドスタイル (CSS - `style.css`)
- ツリーアイテムのベーススタイル（`.tree-item`, `.tree-item:hover`, `.tree-item.active`）を追加します。
- 階層ごとのインデント（再帰的なネスト時の `padding-left` 調整等）を実装します。
- 長いファイル名が枠をはみ出さないように `text-overflow: ellipsis` 等を設定します。

## 検証プラン
1. 起動時にサイドバーに基準ディレクトリの中身が正しく表示されるか確認する。
2. 許可されていない拡張子（例: 画像など）が表示されないことを確認する。
3. フォルダをクリックした際にスムーズに中身が展開・折りたたみされるかテストする。
4. リスト上のテキストファイルをクリックした際に、新しいタブとして正しく開かれるかテストする。
