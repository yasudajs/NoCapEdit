# 実装計画書: ツリーインデントガイドの追加 (Obsidian風)

ファイルツリーの階層関係を視覚的にわかりやすくするため、展開されたフォルダの配下に縦のインデントガイドライン（縦線）を表示するモダンなデザインを導入します。

## 提案する変更

### CSSの調整 (`src/dist/style.css`)

#### [MODIFY] [style.css](file:///c:/work/NoCapEdit/src/dist/style.css)
- `.tree-children` に左ボーダー (`border-left`) を追加し、フォルダの配下に「│」に代わる縦線を描画します。
- これまで暫定的に表示していた「└」の文字ガイドを非表示にし、縦線によるシンプルなインデント表現に切り替えます。
- 縦線が親のフォルダアイコンの真下あたりを通るように、`.tree-children` の `margin-left` および `padding-left` を調整します。

具体的には以下のようなスタイル調整を行います。
```css
.tree-children {
    padding-left: 12px;
    margin-left: 11px; /* 親フォルダアイコン（16px）の中心付近に線を引く */
    border-left: 1px solid var(--border); /* 縦のガイド線 */
}

/* 一時的に追加した └ ガイドを廃止 */
.tree-children .tree-item::before {
    content: none;
}
```

## 検証計画

### 手動検証
1. アプリを起動し、サイドバーを開く。
2. フォルダを展開し、直下のアイテムの左側に薄い縦線が表示されていることを確認する。
3. さらにネストしたフォルダ（例: `bbbb` -> `cccc`）を展開し、縦線が階層の深さに応じて並行して綺麗に描画されていることを確認する。
4. ファイル名やフォルダ名が長くても、アイコンやテキストの位置がずれないこと、全体のレイアウトが崩れないことを確認する。
5. Dark/Soft Dark/Lightテーマのそれぞれにおいて、ガイド線の色が適切（見えやすすぎず、薄すぎない）であることを確認する。
