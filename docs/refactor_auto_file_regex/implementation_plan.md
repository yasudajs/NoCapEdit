# [実装計画書] 連番ファイル名正規表現の不一致修正と一元化 (1-1)

## 概要
`spec.md` (セクション 4.2, 4.3) に定義されている自動生成ファイル名形式 `yyyymmdd_hhmmss.nctx` および連番形式 (`_1`〜`_9`) と、フロントエンド (JS) 側の正規表現判定が食い違っている問題を修正します。
フロントエンド側の `AUTO_FILE_REGEX` 等が `_\d{2}` (2桁想定) となっていたため、連番付きファイル (`_1` 等) が自動生成ファイルと判断されず、タブ表示名フォーマットの不全や自動クリーンアップ処理の漏れが発生していました。

## ユーザー確認済み事項
- **正規表現の連番パターン**: バックエンド実装および仕様（最大 `_9`）と厳密に合わせ、1桁 (`1`〜`9`) のパターン (`(?:_([1-9]))?` または `(?:_(\d))?`) を採用。
- **重複定義の解消 (DRY化)**: `state.js` と `helpers.js` に存在した `AUTO_FILE_REGEX` を `state.js` に一元管理し、共通インポート化する。

## 変更内容

### フロントエンド (JS)

#### [MODIFY] [state.js](file:///c:/work/NoCapEdit/src/dist/js/state.js)
- `AUTO_FILE_REGEX` を `/^\d{8}_\d{6}(_[1-9])?\.nctx$/` に更新（マスターとして一元管理）。

#### [MODIFY] [helpers.js](file:///c:/work/NoCapEdit/src/dist/js/utils/helpers.js)
- 重複定義されている `AUTO_FILE_REGEX` を削除し、`state.js` からインポートするように変更。

#### [MODIFY] [tabs.js](file:///c:/work/NoCapEdit/src/dist/js/ui/tabs.js)
- `formatTabDisplayName` 内のマッチ用正規表現を `/^(\d{4})(\d{2})(\d{2})_(\d{2})(\d{2})(\d{2})(?:_([1-9]))?\.nctx$/` に修正。

   - `yyyymmdd_hhmmss_1.nctx` という連番付きファイルを作成/保存、または既存ファイルとして開き、タブ表示名が `yyyy/mm/dd hh:mm:ss-1` と正しくフォーマットされることを確認。
   - 手動保存モード時、`[yyyy/mm/dd hh:mm:ss-1]` と表示されることを確認。
2. **自動クリーンアップ検証**:
