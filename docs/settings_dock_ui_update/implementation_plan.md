# 設定画面のUI刷新（右ドック形式＆即時反映化）

現在のモーダル（画面中央に表示される）設定ダイアログを廃止し、画面右側からスライドインする「右ドック形式」に変更します。同時に、OKボタンによる確定方式を廃止し、設定項目の変更が即座に反映されるように操作性を向上させます。

## Proposed Changes

### フロントエンドUI（HTML/CSS）

#### [MODIFY] [index.html](file:///c:/work/NoCapEdit/src/dist/index.html)
- 設定ボタン（`#settingsBtn`）の中に、開閉状態を示すためのアイコン（デフォルトの歯車と、非表示状態の✕ボタン）を両方配置し、CSSで表示・非表示を切り替えられるように構造を変更します。
- `dialog-actions` クラスのエリア（OKボタンなどがある箇所）を削除します。

#### [MODIFY] [style.css](file:///c:/work/NoCapEdit/src/dist/style.css)
- `#settingsDialog` に対するスタイルを上書きし、`justify-content: flex-end;` で右寄せにします。
- `.dialog-box` を高さ100%（`100vh`）のパネルにし、右ドック風の見た目に変更します。
- `transform: translateX(100%);` を用いたスライドインアニメーションを追加します。
- `.dialog-content` に `overflow-y: auto; flex: 1;` を設定し、縦スクロールを可能にします。
- 設定ボタン（`#settingsBtn`）が `.open` 状態のとき、内部の「✕アイコン」を表示し「歯車アイコン」を隠すスタイルを追加します。

### フロントエンドロジック（JavaScript）

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
- 設定画面を開く処理を改修し、設定ボタンクリックによるドックの開閉トグル（`.open` クラスの付与・削除）機能に変更します。
- セレクトボックス（保存モードなど）の `change` イベントリスナーを追加し、値の変更時に `saveSettingsDelay()`（または直接的な保存処理と画面への反映）を呼び出すようにします。
- フォルダ選択ボタン（`browseFolderBtn`）クリック後の処理において、フォルダパスが決定された瞬間に保存処理が走るようにします。
- 「OKボタン」のイベントリスナー（`confirmSettingsBtn.onclick`）を削除し、その中の「保存モード切り替えに伴う空ファイルの作成・削除ロジック」を、セレクトボックスの変更イベント内に移譲します。
- 初回起動時（フォルダ未設定時）は、従来通り設定画面を開いた状態（ドックが開いた状態）で起動するように維持します。
