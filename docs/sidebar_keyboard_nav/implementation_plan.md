# 実装計画書 - サイドバーのキーボードナビゲーション & ファイルコピペ機能 (Phase 5)

サイドバー（ファイルエクスプローラ）において、矢印キーによるツリーの操作、エディタとの間でのキーボードによるフォーカス移動、およびファイル・フォルダの複製（コピー＆ペースト）機能を含む高度なキーボード操作を実装します。

## ユーザー確認事項
- **ファイルコピペ・移動時の衝突回避ルール**: 
  ペースト・移動先に同名ファイルが存在する場合、アンダースコア `_copy` は挟まずに、初回重複時から `_01` や `_02` などの連番を付与して自動的に衝突を回避します。コピー元にすでに `_XX` の連番が含まれる場合は、オリジナルのベース名を抽出してインクリメントします。
- **切り取り（Ctrl + X）のキャンセル**:
  `Ctrl + X` でファイルを切り取り中に `Esc` キーを押した場合、切り取り状態がキャンセルされてファイルの半透明化が解除され、クリップボード情報がクリアされます。
- **エディタフォーカスへの復帰キー**: 
  ツリーにフォーカスがある状態で `Esc` キー（切り取り中でない場合）または `Enter` キー（ファイル決定時）が押された際、自動的にテキストエディタにフォーカスが戻ります。

### フロントエンド (JavaScript)

#### [MODIFY] [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
- **状態管理の拡張**:
  - クリップボードを管理するオブジェクト `clipboardState`（`path`, `isDir`, `mode` [copy/cut]）を定義します。
- **キーイベントハンドラの拡張**:
  - 各ツリーアイテム (`div.tree-item`) に `keydown` リスナーを設定し、以下の操作を処理します：
    - **`ArrowDown` / `ArrowUp`**: 現在表示されている（可視の）ツリー項目を走査し、前後へ選択を移動します。
    - **`ArrowRight`**: 選択アイテムがフォルダで閉じている場合、展開します。
    - **`ArrowLeft`**: 選択アイテムがフォルダで展開されている場合、折りたたみます。それ以外（ファイルまたは閉じているフォルダ）の場合、その親フォルダのツリーアイテムを選択します。
    - **`Enter`**: ファイルの場合は開き、エディタにフォーカスを移します。フォルダの場合は展開状態をトグルします。
    - **`Ctrl + N`**: 選択中のフォルダ配下（ファイル選択時は親フォルダ）に新規ファイルのインライン入力を開始します。
    - **`Ctrl + D`**: 選択中のフォルダ配下に新規フォルダのインライン入力を開始します。
    - **`F2`**: 選択中のアイテムの名前変更インライン入力を開始します。
    - **`Ctrl + C`**: 選択中のアイテムパスを `clipboardState`（mode: 'copy'）に記憶し、ステータスバーに通知を表示します。
    - **`Ctrl + X`**: 選択中のアイテムパスを `clipboardState`（mode: 'cut'）に記憶し、ツリーアイテムを半透明化（`cut-pending` クラス付与）します。
    - **`Ctrl + V`**: `clipboardState` の mode に応じて処理します。
      - copy の場合は `copy_file_or_dir` を呼び出し（複製）、cut の場合は `move_file_or_dir` を呼び出し（移動）。移動完了後はクリップボードと半透明状態をクリア。
    - **`Esc`**: エディタへフォーカスを移動します。ただし、現在切り取り中の場合は切り取り状態をキャンセル（半透明解除＆クリップボードクリア）してからエディタへフォーカスします。
- **フォーカス移動 (`Ctrl + E`) の実装**:
  - `main.js` から呼び出せるように、ツリーにフォーカスを当てる関数 `focusSidebarTree()` をエクスポートします。
  - ツリーが閉じている（非表示）状態であれば、自動的にツリーを表示してから Reveal 処理を実行します。
  - **フォーカス時の Reveal（追従）処理**:
    - 現在アクティブなタブが実ファイル（`filePath` が空ではない）の場合：ツリーからそのパスに対応するアイテムを探します。親フォルダが閉じている場合は、上位から順次フォルダを展開し、最終的に対象ファイルを選択状態にしてスクロール表示（`scrollIntoView`）し、フォーカスを当てます。
    - 実ファイルが存在しない場合：ツリーの先頭にある最初の可視アイテムにフォーカスを当てます。
- **グローバル新規作成 (`createItemGlobally`)**:
  - アプリ全体からの新規ファイル/フォルダ作成のための `createItemGlobally(isDir)` 関数を実装・エクスポートします。
  - ツリーが閉じていれば表示させ、選択中フォルダがあればそこ、選択が無ければホームフォルダ直下にインライン入力を追加します。

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/js/main.js)
- アプリ全体の `keydown` リスナーに `Ctrl + E`, `Ctrl + N`, `Ctrl + D` のハンドリングを追加します。
- **`Ctrl + E` 押下時**: `focusSidebarTree()` を実行します。
- **`Ctrl + N` / `Ctrl + D` 押下時**: インライン入力中の INPUT タグ以外の時、`createItemGlobally(isDir)` を実行します。

## 検証計画

### 自動テスト / コンパイルチェック
- `cargo check` を実行して、Rust 側の型定義や `copy_file_or_dir` コマンドの実装にエラーがないことを確認します。
- `cargo build` を実行して、アプリケーションの正常ビルドを確認します。

### 手動検証
1. **フォーカス移動**:
   - エディタ入力中に `Ctrl + E` を押下した際、ファイルツリーの現在開いているファイルにフォーカスが移動し、ツリー上でスクロール・展開されて選択されるか確認。ツリーが閉じている場合は自動で開くことを確認。
   - ツリーフォーカス中に `Esc` または `Enter`（ファイル決定時）で、エディタにフォーカスが戻るか確認。
2. **キーボードナビゲーション**:
   - 矢印キー（上下）で、展開されているフォルダの中身を含めて正しく順次移動ができるか確認。
   - 矢印キー（左右）で、フォルダの展開・折りたたみ、および親フォルダへのフォーカス移動が正しく機能するか確認。
   - `Enter` でファイルが開き、かつフォーカスがエディタへ移るか確認。
3. **新規作成・名前変更ショートカット**:
   - ツリーフォーカス時に `Ctrl + N` / `Ctrl + D` で新規作成、`F2` で名前変更のインライン編集がそれぞれ開始されるか確認。
   - アプリ全体のどこからでも `Ctrl + N` / `Ctrl + D` を押すと、ツリーが自動で開いてホームフォルダ（またはツリー選択フォルダ）宛先に新規作成が開始されるか確認。
4. **コピー＆ペースト**:
   - `Ctrl + C` でファイルをコピーし、別のフォルダ（または同じフォルダ）で `Ctrl + V` を押した際に正しく複製されるか確認。
   - 同名ファイルが存在する場合、`_copy` を挟まずに初回から `_01` 等の連番が自動生成されるか確認。また、連番付きのファイルをさらにコピペした時も `_02` などの正しい連番にインクリメントされるか確認。
5. **切り取り＆貼り付け (移動)**:
   - `Ctrl + X` でファイルを切り取った際、対象が半透明になるか確認。
   - `Esc` を押して切り取りがキャンセルされ、半透明が元に戻るか確認。
   - 別のフォルダで `Ctrl + V` を押した際、ファイルが「移動」し、半透明表示がクリアされるか確認。
   - フォルダの切り取り貼り付けにおいて、子孫階層も含めて正常に移動できるか確認。
   - 移動先に同名ファイルがある場合、自動で連番にリネームして移動できるか確認。
