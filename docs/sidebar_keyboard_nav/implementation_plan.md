# 実装計画書 - サイドバーのキーボードナビゲーション & ファイルコピペ機能 (Phase 5)

サイドバー（ファイルエクスプローラ）において、矢印キーによるツリーの操作、エディタとの間でのキーボードによるフォーカス移動、およびファイル・フォルダの複製（コピー＆ペースト）機能を含む高度なキーボード操作を実装します。

## ユーザー確認事項
- **フォーカス・カーソルの状態遷移仕様**:
  ユーザーが快適にキーボード操作できるように、起動・新規作成・コピペ・削除・ファイル展開といった全操作においてフォーカスの状態遷移を整理・一元管理します（詳細は `spec.md` に記載）。
- **ファイルコピペ・移動時の衝突回避ルール**: 
  ペースト・移動先に同名ファイルが存在する場合、アンダースコア `_copy` は挟まずに、初回重複時から `_01` や `_02` などの連番を付与して自動的に衝突を回避します。
- **切り取り（Ctrl + X）のキャンセル**:
  `Ctrl + X` でファイルを切り取り中に `Esc` キーを押した場合、切り取り状態がキャンセルされてファイルの半透明化が解除され、クリップボード情報がクリアされます。

### フロントエンド (JavaScript)

#### [MODIFY] [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
- **状態管理の拡張とフォーカス一元管理リスナー**:
  - クリップボードを管理するオブジェクト `clipboardState`（`path`, `isDir`, `mode` [copy/cut]）を定義します。
  - **`focusin` / `focusout` によるフォーカス表示状態の一元管理**:
    - ツリー全体（`elements.fileTree`）でフォーカスインされた際、現在フォーカスがある要素をアクティブ選択状態（`.selected` クラス付与、青色表示）に自動更新するイベントリスナーを追加します。
    - フォーカスアウトされた際（次のフォーカス対象がツリーの外である場合）、非アクティブ表示（`.selected-inactive` クラス付与、グレー表示）に自動で切り替えるリスナーを導入します。
    - これにより、コード内の各操作（コピペ、移動、保存等）で個別に青/グレー表示を切り分けるロジックを廃止し、ブラウザフォーカスに完全連動させます。
- **キーイベントハンドラの拡張**:
  - ツリー項目上の `keydown` リスナーに `Ctrl + X` などの処理を実装します。
  - `Ctrl + X`: 選択中のアイテムパスを `clipboardState` に記憶し、ツリーアイテムを半透明化（`cut-pending` クラス付与）します。
  - `Ctrl + V`: 貼り付け完了後、新しく作成/移動された新ファイルのパスをTauriコマンドから受け取り、ツリー上で自動的に Reveal ＆ フォーカス（`focus()`）します（フォーカスイン連動により自動で青色表示されます）。
  - `Esc`: 通常はエディタへフォーカスを移動。切り取り中の場合は切り取りをキャンセル（半透明解除＆クリップボードクリア）した後にエディタへフォーカスします。
- **インライン作成・削除時のフォーカス自動制御**:
  - **新規作成確定時**: ファイル作成完了後は自動でエディタへフォーカスを移動。キャンセル時は作成元の親フォルダにフォーカスを戻します。
  - **削除実行時 (Delete)**: 削除完了後にツリーが再読込されたら、削除されたアイテムの「次の兄弟」「前の兄弟」「親フォルダ」「ツリー先頭項目」の優先順位で自動で新しいアイテムを選択し、フォーカス（`focus()`）を当て直す制御を追加します。
- **描画同期の改善 (`async`/`await` の導入)**:
  - `renderFileTree` を `async` 関数に変更し、展開済み子フォルダをロードする際に `await` で同期的に完了を待機させます。これにより、コピペ完了時にDOMが完全に構築されてから新ファイルのフォーカス処理が確実に走るようにします。

## 検証計画

### 自動テスト / コンパイルチェック
- `cargo check` および `cargo build` を実行して、アプリケーションの正常ビルドを確認します。

### 手動検証
1. **フォーカス移動と遷移**:
   - `Ctrl + E` でファイルツリーへフォーカスが移動し、青色（アクティブ）になるか確認。エディタへカーソルを戻した際、または別をクリックした際にグレー（非アクティブ）に変わるか確認。
   - `Ctrl + N` / `Ctrl + D` で新規作成を開始し、確定したら自動でエディタに入力フォーカスが移り、キャンセルしたら元のフォルダにフォーカスが戻るか確認。
   - `F2` で名前変更を完了/キャンセルした際、そのファイルにフォーカス（青色）が当たった状態に戻るか確認。
2. **コピー・切り取り＆貼り付け**:
   - `Ctrl + X` で切り取ったファイルが半透明になるか確認。
   - `Ctrl + V` で貼り付けた後、新しく配置されたファイルが自動選択されてフォーカス（青色）が当たり、連続した操作がスムーズに行えるか確認。
3. **削除時のフォーカス継続**:
   - ファイルを削除した際、自動で次のファイル（または前のファイル、または親フォルダ）にフォーカスが移動し、キー操作が途切れないか確認。
4. **保存時の選択表示維持**:
   - ツリーからファイルを開いてエディタで保存（自動・手動）した際、ツリーの表示が青色にならず、グレー（非アクティブ選択）のままで維持されるか確認。
