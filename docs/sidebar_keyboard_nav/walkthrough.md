# ウォークスルー - サイドバーのキーボードナビゲーション & ファイルコピペ機能 (Phase 5)

サイドバー（ファイルエクスプローラ）のキーボード操作、フォーカス移動、ショートカット、およびファイルコピー＆ペースト機能を実装しました。

## 変更内容

### 1. バックエンド (Rust)
- [main.rs](file:///c:/work/NoCapEdit/src/main.rs)
  - ディレクトリ配下の全要素を再帰的に複製するヘルパー関数 `copy_dir_all` を実装しました。
  - ファイル/ディレクトリのコピーを行う新しいTauriコマンド `copy_file_or_dir` を追加しました。
    - パスがホームフォルダの配下であることを検証するセキュリティチェックを組み込みました。
    - コピー先に同名ファイルがある場合、`_copy` を挟まずに初回から連番（`_01`, `_02`）で衝突を回避するロジックを実装しました。また、すでに連番が含まれている場合はベース名を抽出してインクリメントするよう改善しました。
    - `tauri::generate_handler` にコマンドを登録しました。
  - ファイル移動コマンド `move_file_or_dir` にも、移動先で同名衝突が起きた場合に連番（`_01`など）で回避する衝突回避ロジックを追加しました。

### 2. フロントエンド (JavaScript)
- [main.js](file:///c:/work/NoCapEdit/src/dist/js/main.js)
  - `Ctrl + E` グローバルショートカットを定義し、ファイルツリーへのフォーカス移動をトリガーするようにしました。
  - アプリ全体（グローバル）で機能する `Ctrl + N`（新規ファイル） / `Ctrl + D`（新規フォルダ）ショートカットを追加し、エディタ上やツリー外からでも新規作成を実行できるようにしました。
- [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
  - アプリ全体およびエディタからファイルツリーへのフォーカスを移動する `focusSidebarTree()` 関数を実装・エクスポートしました。
    - ツリーが閉じている（非表示の）状態で `Ctrl + E` が押された場合、自動でサイドバーを展開する制御を追加しました。
    - アクティブタブの実ファイルが存在する場合、その親階層のフォルダを自動的に上から順次展開し、対象アイテムにスクロールしてフォーカスする（Reveal）機能を組み込みました。
    - 開いている実ファイルがない場合は、ツリーの先頭項目にフォーカスします。
  - アプリ全体の新規作成キーから呼び出される `createItemGlobally(isDir)` 関数を実装・エクスポートしました。
    - キー押下時にツリーが閉じている場合は自動でツリーを開きます。
    - ツリーの項目が選択されていればその配下に、選択が外れている場合はホームフォルダ直下にインライン入力で新規作成を開始します。
  - 各ツリー項目（`div.tree-item`）に `keydown` リスナーを登録し、キーボードナビゲーションを実装しました。
    - `ArrowUp` / `ArrowDown` : ツリー内の現在可視状態の項目を前後へ移動。
    - `ArrowRight` : 閉じているフォルダを展開、または展開済みフォルダの最初の子要素へ移動。
    - `ArrowLeft` : 展開されているフォルダを閉じる、または親フォルダのアイテムへ選択を移動。
    - `Enter` : ファイルを開く（同時にエディタへフォーカスを移動）、またはフォルダの展開/折りたたみのトグル（Spaceキーは操作から場外）。
    - `Esc` : エディタにフォーカスを戻す。
  - ツリーフォーカス時に動作する操作ショートカットを追加しました。
    - `Ctrl + N` / `Ctrl + D` : 選択中のフォルダ配下にインライン入力状態の新規ファイル/新規フォルダを作成。
    - `F2` : 選択項目をインライン名前変更状態にする。
    - `Ctrl + C` / `Ctrl + V` : 選択項目のパスをメモリに記憶し、選択フォルダ配下に複製（Tauriの `copy_file_or_dir` コマンドの呼び出しとツリーの再描画・選択クリア）。
