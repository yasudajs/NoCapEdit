# タブ表示名のフォーマット対応（日時表示・拡張子非表示化）

タブに表示されるファイル名を、自動生成されたファイルについては日時形式 `yyyy/mm/dd hh:mm:ss` に変換し、それ以外のファイルについては拡張子を非表示にする対応を行います。

## ユーザーレビューが必要な項目

- ありません。合意いただいた方針（同名衝突のインデックスは `-1` のようにハイフンで繋ぎ、その他のファイルは拡張子を隠す）に基づいて実装します。

## 提案する変更

### フロントエンド (JavaScript)

`src/dist/main.js` にて、タブの表示名を決定するヘルパー関数を導入し、レンダリング時に適用します。

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
- 自動生成ファイル名（`20260709_222843.nctx` や `20260709_222843_01.nctx` など）を `2026/07/09 22:28:43` や `2026/07/09 22:28:43-1` に変換するヘルパー関数 `formatTabDisplayName(fileName)` を実装します。
- それ以外のファイル（例: `memo.txt`）については、最後の拡張子（最後の `.` 以降の文字列）を取り除いて表示します。
- `renderTabs()` の中で `nameEl.textContent = tab.fileName;` となっている部分を `nameEl.textContent = formatTabDisplayName(tab.fileName);` に変更します。

## 検証計画

### 手動検証
1. アプリを起動し、新規に作成された自動生成タブが `2026/07/09 22:28:43` のような日時形式で表示されていることを確認する。
2. 同時に複数の新規タブを作成し、同名（同秒）ファイルの回避処理が走り `20260709_222843_01.nctx` のようなファイルが生成された場合に、タブ名が `2026/07/09 22:28:43-1` になっていることを確認する。
3. `Ctrl + S` もしくは設定等から「別名で保存」を行い、例えば `test_memo.txt` や `document.md` として保存した際に、タブ名がそれぞれ `test_memo` や `document` に切り替わり、拡張子が非表示になっていることを確認する。
