# Ctrl + E ショートカットのトグル挙動実装計画

## 概要

現状の `Ctrl + E` は「ファイルツリーを開いてフォーカスを移す」単一方向の動作のみですが、これを拡張し「ファイルツリーとエディタを行き来するトグルショートカット」にします。
さらに、ファイルツリー上でファイルを選択中に `Ctrl + E` を押した際、単にエディタに戻るだけでなく、そのファイルを自動的に開く（既に開かれている場合はそのタブをアクティブにする）ことで、キーボード主体のシームレスな操作体験を実現します。

## 状態と動作の整理

| 現在の状態 | 操作 | 対象 | 結果の動作 |
| :--- | :--- | :--- | :--- |
| **ツリー閉・エディタフォーカス** | `Ctrl + E` | (なし) | ツリーを開き、ツリーへフォーカス |
| **ツリー開・エディタフォーカス** | `Ctrl + E` | (なし) | ツリーへフォーカスを移す（ツリーは開いたまま） |
| **ツリー開・ツリーフォーカス** | `Ctrl + E` | **ファイル** | その**ファイルを開き**、ツリーを閉じ、エディタへフォーカス |
| **ツリー開・ツリーフォーカス** | `Ctrl + E` | **フォルダ** | ツリーを閉じ、エディタへフォーカス |
| **ツリー開・ツリーフォーカス** | `Esc` | ファイル/フォルダ | **ツリーは開いたまま**、エディタへフォーカス |

## Proposed Changes

### フロントエンド (JavaScript)

#### [MODIFY] [ui/sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)

1.  **`focusSidebarTree()` の改修**
    *   関数内で現在のアクティブ要素（`document.activeElement`）を確認し、**既にファイルツリー内にフォーカスがあるか**を判定する分岐を追加。
    *   **ファイルツリー内にフォーカスがある場合**：
        *   `selectedElement` からファイルパスおよび `isDir`（フォルダかどうか）を取得。
        *   もし対象が**ファイル**であれば、既存の `Enter` キー押下時と同様にファイルを開く処理（またはタブをアクティブにする処理）を呼び出す。
        *   サイドバーを閉じる処理（`appState.sidebarVisible = false`、および各DOM要素への `.hidden` クラス追加など）を実行。
        *   `elements.editor.focus()` でエディタにフォーカスを戻す。
    *   **ファイルツリー外にフォーカスがある場合（現状のまま）**：
        *   ツリーを開き、前回選択アイテム（または最上部アイテム）にフォーカスを当てる。

2.  **`Enter`キー押下時処理のリファクタリング（任意）**
    *   ファイルのオープン処理（既存タブのチェックなど）が `keydown` イベントリスナー内に直接記述されている場合、`Ctrl + E` からも呼び出しやすいように小規模な関数抽出を行う（`openFileAndFocusEditor` のような形）。

## Verification Plan

### Manual Verification
以下の手順で手動テストを実施し、キーボードのみで意図した挙動になることを確認する。

1.  **トグル動作（フォルダ）の確認**
    *   エディタフォーカス状態で `Ctrl + E` を押す → ツリーが開きフォーカスが当たる。
    *   矢印キーでフォルダを選択する。
    *   再度 `Ctrl + E` を押す → ツリーが閉じ、エディタにフォーカスが戻る。
2.  **トグル動作（ファイル・新規タブ）の確認**
    *   `Ctrl + E` でツリーを開く。
    *   矢印キーで**まだ開いていないファイル**を選択する。
    *   `Ctrl + E` を押す → そのファイルが新しいタブで開き、ツリーが閉じてエディタにフォーカスが戻る。
3.  **トグル動作（ファイル・既存タブ）の確認**
    *   すでにいくつかのタブが開かれている状態にする。
    *   エディタから `Ctrl + E` でツリーを開く。
    *   矢印キーで**すでに別のタブとして開かれているファイル**を選択する。
    *   `Ctrl + E` を押す → 既存の該当タブがアクティブに切り替わり、ツリーが閉じてエディタにフォーカスが戻る。
4.  **Esc キーの動作確認**
    *   `Ctrl + E` でツリーを開き、ファイルまたはフォルダを選択する。
    *   `Esc` キーを押す → **ツリーは開いたまま**、エディタにフォーカスが戻る。
    *   その状態で再度 `Ctrl + E` を押す → ツリーのフォーカス状態が復元される。
