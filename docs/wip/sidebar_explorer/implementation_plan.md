# サイドバー（ファイルエクスプローラ）機能の実装計画（検討中）

NoCapEdit にObsidian風の左サイドバーを追加し、ホームフォルダ以下のファイル・フォルダを操作できる簡易エクスプローラ機能を実装する。

> [!NOTE]
> 本ドキュメントは検討中の実装計画書です。仕様確定後に作業ブランチを作成し、実装に移行します。

---

## ユーザー要件サマリ

| 項目 | 仕様 |
|---|---|
| 表示対象 | ホームフォルダ以下すべて（ツリー構造）。対応ファイル: `.nctx`, `.txt`, `.csv` 等の一般テキスト系を優先 |
| ファイル操作 | クリック選択、ダブルクリックでタブ追加、ファイル/フォルダの作成・削除・リネーム |
| 移動 | ドラッグ＆ドロップでファイル移動 |
| 開閉 | ボタンでトグル（設定パネルのようなオーバーレイではなく、エディタと横並びで表示） |
| 幅調整 | 境界線のドラッグでリサイズ可能 |
| ソート | ファイル名順（デフォルト）、ドロップダウンで変更、昇順/降順の切り替えボタン |
| 初期状態 | 起動時はサイドバーを閉じた状態にする |
| 削除 | 通常削除はゴミ箱へ移動（確認ダイアログあり）。`Shift + Del` で完全削除（確認ダイアログあり） |
| バージョン | 未定 |

## 開発対象ファイルと変更内容

---

### Rustバックエンド

#### [MODIFY] main.rs

**① 新規Tauriコマンド**

| コマンド名 | 説明 |
|---|---|
| `list_directory(path)` | 指定フォルダ直下の全ファイル・サブフォルダ一覧を返す（名前・パス・種別・更新日時・サイズ） |
| `create_folder(path)` | 指定パスにフォルダを作成する |
| `create_file_in_dir(dir_path)` | 指定フォルダ内に `yyyymmdd_hhmmss.nctx` 形式でファイルを新規作成する |
| `rename_entry(old_path, new_name)` | ファイルまたはフォルダをリネームする |
| `move_entry(src_path, dst_dir)` | ファイルまたはフォルダを指定フォルダへ移動する |
| `trash_entry(path)` | ファイルまたはフォルダをOSのゴミ箱へ移動する（`trash` クレートを使用） |
| `delete_entry_permanent(path)` | ファイルまたはフォルダを完全削除する（`Shift+Del` 時） |
| `start_watch_directory(path)` | 指定フォルダの変更監視を開始し、変更があればTauriイベント `directory-changed` を発行する |

> [!NOTE]
> **`trash` クレートの追加について**
> ゴミ箱への移動には `trash` クレート（`Cargo.toml` への追記）が必要です。
> - `trash = "5"` を `[dependencies]` に追加

**② `AppSettings` への新規フィールド追加**

| フィールド | 型 | デフォルト | 説明 |
|---|---|---|---|
| `sidebar_visible` | `bool` | `false` | サイドバーの表示/非表示 |
| `sidebar_width` | `u32` | `220` | サイドバーの幅（ピクセル） |
| `sidebar_sort_key` | `String` | `"name"` | ソートキー（`"name"` / `"modified"` / `"size"`） |
| `sidebar_sort_asc` | `bool` | `true` | 昇順(`true`) / 降順(`false`) |

**③ ファイル変更監視**
- `notify` クレート（Tauriが内部使用）または Tauri v1 の `tauri-plugin-fs-watch` を使ってホームフォルダを監視
- ファイル追加・削除・リネームが検知されたら `directory-changed` イベントをフロントエンドに発行
- 困難な場合は案A（3〜5秒ポーリング）にフォールバックする

#### [MODIFY] main.js

以下の機能を追加する：

**① サイドバーの開閉・リサイズ**
- `sidebar-toggle-btn` クリックでサイドバーの表示/非表示をトグル
- リサイズハンドルのマウスドラッグで `sidebar` の幅を動的変更
- 表示状態・幅を `save_settings` で `config.json` に永続化

**② ファイルツリーの構築**
- `list_directory(homeFolder)` を呼び出しツリーデータを取得
- フォルダはクリックで展開/折りたたみ（再帰的な `list_directory` 呼び出し）
- ファイルのシングルクリック → 選択ハイライトのみ
- ファイルのダブルクリック → 既存の `openFileInNewTab()` と連携してタブを追加

**③ ソートコントロール**
- ドロップダウン選択（名前 / 更新日時 / サイズ）
- `↑` / `↓` ボタンで昇順/降順を切り替え
- ツリーを即座に再描画。状態を `config.json` に永続化

**④ コンテキストメニュー（右クリック）**

| クリック対象 | メニュー項目 |
|---|---|
| ファイル | 開く / リネーム / 削除（ゴミ箱）/ 完全削除 |
| フォルダ | 新規ファイル / 新規フォルダ / リネーム / 削除（ゴミ箱）/ 完全削除 |
| 空白エリア | 新規ファイル / 新規フォルダ |

**⑤ インラインリネーム**
- リネーム選択時、ツリー内のラベルを `<input>` に差し替えてインライン編集
- `Enter` で確定 → `rename_entry` コマンド呼び出し
- `Escape` でキャンセル

**⑥ ドラッグ＆ドロップ（ファイル移動）**
- HTML5 Drag and Drop APIを使用
- ドラッグ中はアイテムをハイライト表示
- フォルダへのドロップで `move_entry(src, dstDir)` を呼び出して移動
- 移動完了後にツリーを更新（再描画）

**⑦ 削除処理**
- 通常削除（右クリックメニュー or `Delete` キー）
  1. Tauri の `dialog::confirm` で「ゴミ箱に移動しますか？」を確認
  2. 確認OKで `trash_entry(path)` を呼び出す
- 完全削除（右クリックメニュー or `Shift + Delete` キー）
  1. Tauri の `dialog::confirm` で「完全に削除しますか？（元に戻せません）」を確認
  2. 確認OKで `delete_entry_permanent(path)` を呼び出す

**⑧ ファイル変更の自動検知**
- バックエンドから `directory-changed` イベントを購読し、受信時にツリーを自動更新
- ファイル監視が不可能な場合は、サイドバー表示中のみ一定間隔でポーリングにフォールバック

## 対応するファイル種別

| 拡張子 | 説明 | タブで開く |
|---|---|---|
| `.nctx` | NoCapEdit専用形式 | ✅ |
| `.txt` | 一般テキスト | ✅ |
| `.csv` | CSVファイル | ✅（テキストとして） |
| `.md`, `.ncmd` | Markdown | ✅ |
| 上記以外 | 非対応形式 | ❌ ステータスバーにエラー通知 |

---

## 設定画面への影響

設定画面（右ドック）への新規項目の追加は不要。  
ただし `save_settings` / `get_settings` コマンドの引数に  
`sidebar_visible`, `sidebar_width`, `sidebar_sort_key`, `sidebar_sort_asc` の4項目が増える。
