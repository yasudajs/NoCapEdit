# 仕様と実装の乖離修正 マスタープラン

`spec.md` と現在のソースコード実装との間の乖離を一つずつ解決するための全体計画書です。

## 1. クリティカルな不具合・実装漏れ

- **[x] 1-1. 連番ファイル名の正規表現の不一致（セクション 4.2, 4.3）**
  - **乖離内容**: JS側（`state.js`, `helpers.js`, `tabs.js` 等）の正規表現 `AUTO_FILE_REGEX` が `(_\d{2})?` と2桁の連番を前提としている。フェーズ1-2/1-3のバックエンド修正（ゼロ埋め廃止）が反映されていない。
  - **方針**: JS側の正規表現を `(_\d)?` に修正し、自動クリーンアップやタブ表示が正しく動作するように実装を修正する。

- **[x] 1-2. 無保存モード時の手動保存における連番回避の欠如（セクション 4.4）**
  - **乖離内容**: 無保存モードで同秒数で複数回手動保存した場合に、連番回避ロジック（Rustの `create_and_save_file`）を通らずにJS側で直接保存を叩いており、既存ファイルを上書き破壊してしまう恐れがある。
  - **方針**: 手動保存時にも連番回避が効くよう実装を修正する（Rust側への委譲かJS側での存在チェック追加）。

## 2. 仕様書と実装の食い違い（方針すり合わせ中）

- **[ ] 2-1. 「安全保存方式（アトミック保存）」の未実装（セクション 4.4）**
  - **乖離内容**: 仕様書は一時ファイル経由のアトミック保存だが、実装はファイル監視イベント発火を防ぐため直接上書き（`fs::write`）。
  - **方針**: 【ユーザー確認待ち】仕様書を書き換えて直接上書きを正とするか、ファイル監視を高度化してアトミック保存に対応させるか。

- **[ ] 2-2. 設定値・定数の一元管理（セクション 4.8）**
  - **乖離内容**: 仕様書には `constants.rs` で一元管理するとあるが、現在は `main.rs` の先頭に定数が直書きされている。
  - **方針**: 【ユーザー確認待ち】仕様書通りに `constants.rs` を作って分離するか、現状維持で仕様書を修正するか。

- **[ ] 2-3. ブラウザ標準コンテキストメニューの無効化の副作用（セクション 4.13）**
  - **乖離内容**: 全体の右クリックを無効化した結果、エディタ（textarea）上でも標準の右クリックメニュー（コピー・貼り付け等）が出なくなっている。
  - **方針**: 【ユーザー確認待ち】ユーザビリティのため、textarea上のみ標準メニューを許可するように実装・仕様を変更するか。

## 3. 軽微な実装漏れ

- **[x] 3-1. 禁止文言「準備完了」の残留（セクション 4.5）**
  - **乖離内容**: 仕様書で禁止されている曖昧な文言が `main.js` 等に `updateStatus('準備完了');` として残っている。また、日本語テキストが直接ハードコードされており、多言語対応（`i18n.js`）ルールに反していた。
  - **方針（完了）**: 
    - `i18n.js` に `status_ready_auto`（"保存準備完了"）および `status_ready_manual`（"※Ctrl+Sで保存できます"）を定数化。
    - 手動保存モード時は `updateStatus` 関数側のプレフィックス自動付与ロジック（`[手動保存モード] `）を活用し、DRY原則に従い二重表示を回避するようリファクタリング。
    - `main.js`, `settings.js`, `tabs.js` の呼び出し箇所を、保存モードを判定して `t('...')` を渡す形へ修正。

- **[x] 3-2. ショートカット無効化の漏れ（セクション 4.12）**
  - **乖離内容**: `Ctrl + Shift + R` と `Ctrl + F5` の無効化リスナーが漏れている。
  - **方針（完了）**: JS側（`main.js`）のキーボードリスナーに対象キー（`Ctrl+Shift+R`, `Ctrl+F5`, `Ctrl+Shift+F5`）を追加して無効化し、`SHORTCUTS.md` にも無効化ショートカットのセクションを明記。

- **[x] 3-3. `Ctrl + E` 時のアクティブタブ自動フォーカス（セクション 4.10）**
  - **乖離内容**: ツリー表示トグル時、仕様にある「現在アクティブなタブの実ファイルを自動展開してフォーカスする」連携が未実装。
  - **方針（完了）**: `sidebar.js` の `focusSidebarTree()` ロジックを改修し、現在アクティブなタブの実ファイルパスを元にツリー項目を探索・選択し、閉じている親フォルダを自動展開してスクロール・フォーカスするよう実装。
