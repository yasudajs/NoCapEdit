# v0.2 リファクタリングおよび機能分離 実装計画

本計画は、v0.2系のコードベースを見直し、基本機能（v0.1と共通するコア部分）と追加機能（v0.2特有のサイドバーなど）の分離を行うためのものです。これにより、将来的にv0.1系（`master`）をこの共通ベースにリファクタリングする際の手間とリスクを最小化します。

## 🎯 目標
- `main.js` などのコアファイルにハードコードされているv0.2固有のロジックを、専用のモジュールファイルに分離（カプセル化）する。
- 基本機能と追加機能の「関心の分離」を行い、v0.2系自体の保守性を向上させる。
- v0.1系をこの構造に同期する際、**「追加モジュールをインポートしないだけで動作する」**ようなクリーンな共通ベースを構築する。

---

## 🛠️ 作業フェーズの分割

リスクを抑え、少しずつ動作確認しながら進めるため、作業を以下の細かいフェーズに分割して実行します。

### フェーズ 1: サイドバーロジックの完全カプセル化
現在 `main.js` 内に散らばっているサイドバー（`v0.2` 固有機能）の処理を `ui/sidebar.js` に集約・隠蔽します。

- **対象**: `src/dist/js/main.js`, `src/dist/js/ui/sidebar.js`
- **作業内容**:
  - `main.js` の `init()` にあるサイドバー表示状態（幅や `hidden` の切り替え）の制御ロジックを、`sidebar.js` の `applySidebarState(appState)` のような関数に移動する。
  - サイドバーが完全に独立したモジュールとして動作し、`main.js` からは初期化関数を呼ぶだけの状態にする。

### フェーズ 2: キーボードショートカットの分離
`main.js` にベタ書きされているショートカット処理（`Ctrl + E`, `Ctrl + N` など）を分離し、機能ごとに管理します。

- **対象**: `src/dist/js/main.js`, `src/dist/js/ui/sidebar.js` (または新規 `ui/shortcuts.js`)
- **作業内容**:
  - `window.addEventListener('keydown')` 内の処理を整理する。
  - サイドバー操作（`e`, `n`, `d`）に関するショートカット判定ロジックを `sidebar.js` 側に委譲するか、イベントディスパッチャを用いてメインループから切り離す。
  - v0.1系と共通のショートカット（ズームや保存）のみを基本ロジックとして残す。

### フェーズ 3: ファイルシステム監視 (Watcher) の分離
`main.js` の `file-system-changed` イベント（ファイル作成・削除の検知）内にある、タブの更新とサイドバーのツリー更新ロジックを分離します。

- **対象**: `src/dist/js/main.js`, `src/dist/js/ui/sidebar.js`, `src/dist/js/ui/tabs.js`
- **作業内容**:
  - 現在 `main.js` (L315〜L368) で行われている「サイドバーの特定のフォルダを特定して `loadDirectory` を呼ぶ」複雑なロジックを、`sidebar.js` の `handleFileSystemChangeForSidebar(paths)` に切り出す。
  - `main.js` では「タブのリネーム・クローズ」等の基本処理のみを行い、その後 `sidebar.js` のハンドラを呼び出すだけのシンプルな構造にする。

### フェーズ 4: UIイベントリスナーと状態管理の整理
各種設定変更（テーマ、フォント、文字数カウントモード等）のイベントリスナー登録を整理します。

- **対象**: `src/dist/js/main.js`, `src/dist/js/ui/settings.js`
- **作業内容**:
  - `main.js` の `setupUIEventListeners` が肥大化しているため、設定に関するイベント登録（`elements.tabBehaviorSelectModal` など）は `settings.js` の初期化関数（例: `setupSettingsEventListeners()`）に委譲する。
  - タブの自動スクロール処理（`ui/tabs.js`）なども、責務が明確になるようコードを整理する。
