# ステータスバー表示の改善計画 (新規作成および初回保存時)

新規ウィンドウ・新規タブ作成直後、まだディスク上にファイルが作成されていない遅延保存の待機状態において、ステータスバーの表示が「〜を作成」となっている問題を解消し、実際のファイル作成タイミングおよび保存モードに合わせた適切な表示に改善します。

さらに、編集や保存に伴ってファイル名がステータスバーから消えてしまう問題を解消し、ファイル実体が存在する場合は「ファイル名 - 状態」を常時表示するように統一します。また、タブ切り替え時にはそのタブの現在の状態を正確に復元します。

## ユーザーレビュー要求事項

> [!IMPORTANT]
> 1. **初期状態（ファイル未作成）の表示**:
>    - 自動保存モード時：`保存準備完了`
>    - 手動保存モード時：`[手動保存モード] ※Ctrl+Sで保存できます`
> 2. **ファイル実体が存在する状態の表示**:
>    - `[接頭辞] [ファイル名] - [状態]` の形式で表示を統一します。
>      - `[接頭辞]` : 自動保存モード時はなし。手動保存モード時は **`[手動保存:Ctrl+S] `** とします。
>      - `[状態]` : `編集中` / `保存中...` / `保存済み` / `保存失敗` 等（※曖昧な「準備完了」表示は廃止）。
>    - ※初回保存が完了した瞬間のみ、状態表示の代わりに `[ファイル名] を作成`（手動保存モード時は `[手動保存:Ctrl+S] [ファイル名] を作成`）と表示します。
> 3. **タブ切り替え時のステータス復元**:
>    - 他のタブへ切り替えた（アクティブにした）際は、そのタブの現在の状態（保存済み / 編集中 / 未作成の準備完了など）を自動判別し、ステータス表示を正確に復元します。

## 提案する変更

### フロントエンド (UIロジック)

---

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
ステータスバー表示の切り替えおよび保存時のメッセージ制御ロジックを修正します。

##### 1. `updateStatus` 関数のシグネチャ拡張と接頭辞バイパス機能の追加
- `updateStatus(message, status = 'normal', bypassPrefix = false)` に変更します。
- `bypassPrefix` が `true` の場合、手動保存モードであっても自動的な `[手動保存モード]` 接頭辞の付与を行わず、引数 `message` をそのまま出力します。これで呼び出し側がカスタム接頭辞（`[手動保存:Ctrl+S]` など）を制御可能にします。

##### 2. タブ個別ステータス表示用ヘルパー `updateTabStatus` の新規導入
- 指定されたタブのファイルパス有無および保存状態（`isDirty`, `isSaving`）に応じた表示文言を組み立てる共通関数を定義します。
- 引数 `state` が `null` の場合は、タブオブジェクトの内部状態から自動的に状態文字列（`編集中` / `保存中...` / `保存済み`）を判別して復元します。
```javascript
function updateTabStatus(tab, state = null, statusType = 'normal') {
    if (!tab) return;
    
    // state引数が省略された場合は、現在のタブの状態から動的に判定して復元する
    let targetState = state;
    if (!targetState) {
        if (tab.isSaving) {
            targetState = '保存中...';
        } else if (tab.isDirty) {
            targetState = '編集中';
        } else {
            targetState = '保存済み';
        }
    }
    
    if (!tab.filePath) {
        // ファイル未作成（初期状態）
        if (appState.saveMode === 'manual') {
            updateStatus('※Ctrl+Sで保存できます', statusType);
        } else {
            updateStatus('保存準備完了', statusType);
        }
    } else {
        // ファイル作成済み
        let prefix = '';
        if (appState.saveMode === 'manual') {
            prefix = '[手動保存:Ctrl+S] ';
        }
        updateStatus(`${prefix}${tab.fileName} - ${targetState}`, statusType, true);
    }
}
```

##### 3. 各イベントハンドラでのステータス表示呼び出しの差し替え
- **新規タブ作成時 (`createNewTab` 関数)**:
  - `updateTabStatus(tab)` を呼ぶように修正します（以前のように `〜 を作成` と表示するのを廃止）。
- **タブ切り替え時 (`switchTab` 関数)**:
  - 従来の `updateStatus(tab.fileName + ' - 準備完了')` を `updateTabStatus(tab)` に差し替えます（状態の自動復元）。
- **テキスト編集時 (`handleEditorInput` または入力監視箇所)**:
  - 従来の `updateStatus('編集中...')` を `updateTabStatus(tab, '編集中')` に差し替えます。
- **保存処理時 (`autoSave` および `triggerManualSave` 関数)**:
  - 保存実行中： `updateTabStatus(tab, '保存中...', 'saving')`
  - 保存完了時：
    - 初回保存（保存処理前に `!tab.filePath`）だった場合：
      - 接頭辞を考慮して `updateStatus(tab.fileName + ' を作成', 'saved', false)`（手動保存モード時は自動で `[手動保存:Ctrl+S] ` を付与、または明示的に指定）を表示。
    - 2回目以降の保存の場合：
      - `updateTabStatus(tab, '保存済み', 'saved')`
  - 保存失敗時：
    - `updateTabStatus(tab, '保存失敗', 'error')`

## 検証計画

### 手動検証
1. **自動保存モードでの動作確認**
   - アプリを起動、または `+` ボタンで新規タブを作成した際、ステータスバー右下に `保存準備完了` と表示されること。
   - 文字を入力した際、ステータスバーが `編集中` になるのではなく、ファイルは未作成のため `保存準備完了` のままであること。（※最初の保存が走るまではファイル名表示なし）
   - 3秒後の自動保存が実行されたタイミングで `20260711_224758.nctx を作成` と表示されること。
   - その後、さらに編集を加えて自動保存された際は `20260711_224758.nctx - 編集中` と表示され、保存完了後は `20260711_224758.nctx - 保存済み` と表示されること。
2. **手動保存モードでの動作確認**
   - 設定から手動保存（無保存モード）に切り替え、新規タブを作成した際、ステータスバー右下に `[手動保存モード] ※Ctrl+Sで保存できます` と表示されること。
   - 文字を入力した際も `[手動保存モード] ※Ctrl+Sで保存できます` のままであること。
   - `Ctrl+S` を押して手動保存を実行したタイミングで `[手動保存:Ctrl+S] 20260711_224758.nctx を作成` と表示されること。
   - 再度編集した際は `[手動保存:Ctrl+S] 20260711_224758.nctx - 編集中` になり、再度 `Ctrl + S` で保存した際は `[手動保存:Ctrl+S] 20260711_224758.nctx - 保存済み` と表示されること。
3. **タブ切り替え時の動作確認**
   - 未保存の新規タブと、既に保存された既存タブを切り替えた際、それぞれの状態に応じたステータスメッセージ（例：`保存準備完了` ⇔ `20260711_224758.nctx - 保存済み`）が正確に復元されること。
   - 手動保存モードで編集中のタブと他のタブを切り替えた際、戻ってきた時に `[手動保存:Ctrl+S] 20260711_224758.nctx - 編集中` が正確に復元されること。
