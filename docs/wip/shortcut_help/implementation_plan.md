# ヘルプ機能（ショートカットキー一覧のモーダル表示）実装計画

アプリ利用中にショートカットキーを手軽に確認できるよう、`F1` キーまたは `Ctrl + /` キーを押すことで画面上にスタイリッシュなオーバーレイ（モーダルダイアログ）を表示し、ショートカットキー一覧を表示する機能を実装します。

将来の多言語対応に配慮し、表示データは直接ハードコードせず [i18n.js](file:///c:/work/NoCapEdit/src/dist/i18n.js) に定義し、JavaScript 側で動的にUIを構築する設計とします。

---

## ユーザーレビューが必要な項目

> [!IMPORTANT]
> **ヘルプデータの定義場所について**
> [i18n.js](file:///c:/work/NoCapEdit/src/dist/i18n.js) に配列・オブジェクト形式の構造化データとしてショートカットキー一覧を定義します。キーの文言（`Ctrl + S` 等）と説明文（`手動保存` 等）をペアで管理し、将来的に多言語（英語など）へ切り替えた際にもUI生成ロジックを変更することなく翻訳が適用されるようにします。

---

## 提案する変更内容

### フロントエンド

#### [MODIFY] [index.html](file:///c:/work/NoCapEdit/src/dist/index.html)
* ヘルプを表示するための `<dialog>` 要素を本体の末尾に追加します。
* モーダルの基本的な骨組み（タイトル、閉じるボタン、コンテンツ挿入用のコンテナ）を定義します。

#### [MODIFY] [style.css](file:///c:/work/NoCapEdit/src/dist/style.css)
* ヘルプモーダル用のCSSクラスを追加します。
* プレミアムなデザインにするため、以下のようなスタイルを適用します。
  * 背景のぼかし効果（`backdrop-filter: blur(...)` を用いたグラスモーフィズム）
  * ダークモード/ライトモードに応じた配色
  * ショートカットキー（`kbd` 要素など）のスタイリッシュなバッジ風デザイン
  * カテゴリごとのテーブルレイアウトおよびレスポンシブ対応（はみ出し時のスクロール）

#### [MODIFY] [i18n.js](file:///c:/work/NoCapEdit/src/dist/i18n.js)
* 辞書データ `DICT.ja` 内にヘルプ用のテキストラベルおよび、カテゴリ別に分類したショートカットキーの構造化データを追加します。
* 例：
  ```javascript
  DICT.ja = {
      // ... 既存データ ...
      help_modal_title: "キーボードショートカット一覧",
      help_modal_close: "閉じる",
      
      shortcut_cat_editor: "エディタ操作",
      shortcut_cat_tabs: "タブ操作",
      shortcut_cat_tree: "ファイルツリー操作",
      
      shortcut_list: [
          { cat: "shortcut_cat_editor", key: "Ctrl + S", desc: "手動保存 (自動保存モード時も即時保存)" },
          { cat: "shortcut_cat_editor", key: "Ctrl + + / -", desc: "文字サイズの拡大・縮小" },
          { cat: "shortcut_cat_tabs", key: "Ctrl + Tab", desc: "右隣のタブへ移動" },
          // ... 既存の docs/SHORTCUTS.md に基づく全ショートカット ...
      ]
  };
  ```

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/js/main.js)
* キー監視およびモーダル制御処理を追加します。
  * `keydown` イベントリスナーにて `F1` または `Ctrl + /` (あるいは `Ctrl + ?`) の入力をトラップします。
  * ヘルプ表示時に、`i18n.js` から取得した `shortcut_list` を走査し、カテゴリごとに整理されたHTMLテーブルを動的に組み立ててモーダル内に挿入します。
  * モーダルを開く処理 (`showModal()`) および閉じる処理 (`close()`) を定義し、閉じるボタンや背景クリック、`Esc` キーでのクローズをハンドリングします。

## 検証計画

### 自動テスト
* 現時点ではUI表示に関する自動テストはないため、コンパイル/ビルドが正常に通ることを確認します。

### 手動検証
1. **キーによる開閉確認**
   * アプリ起動中、どのフォーカス状態（エディタ、ファイルツリー、設定画面など）からでも `F1` または `Ctrl + /` を押した際にヘルプモーダルが表示されること。
   * モーダル表示中に `Esc` キー、または閉じるボタンをクリックした際にモーダルが閉じること。
2. **UIと多言語データの整合性**
   * 表示されるショートカットキーとその説明が、[SHORTCUTS.md](file:///c:/work/NoCapEdit/docs/SHORTCUTS.md) と一致していること。
   * カテゴリごとに綺麗に分類されて表示されていること。
3. **ビジュアル・操作性の確認**
   * 背景が適度にぼかされ、エディタの内容が見えつつモーダルに集中できるデザインになっていること。
   * 高さが足りない画面サイズの場合、モーダル内が適切にスクロール可能であること。
