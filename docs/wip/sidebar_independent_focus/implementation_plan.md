# 実装計画書 - ツリーの独立動作化とフォーカス復元 (Phase 5.1)

エディタ（タブ管理）とファイルツリー（サイドバー）のフォーカスおよび選択状態を完全に独立させ、不要な自動連動（Reveal）を廃止します。`Ctrl + E` によるフォーカス切り替え時は、ツリー側で「最後にアクティブだった項目」へ正確にフォーカスを戻します。

## ユーザー確認事項
- **`Ctrl + E` 押下時の挙動**: 
  - アクティブなファイルへの自動 Reveal を廃止します。
  - ツリーの `selectedPath` に一致する要素があれば、その項目に直接フォーカスします。
  - 初回起動時など一度も選択がない場合は、ツリー最上部（「nce」ホームフォルダ自体）を選択・フォーカスします。
- **タブ切り替え時の非連動**: 
  - エディタ側でタブを切り替えても、ツリーの表示や選択状態は一切書き換わらず、直前の状態（スクロール位置、フォルダ展開状態、グレー背景の選択）を維持します。

## 提案する変更

### フロントエンド (JavaScript)

#### [MODIFY] [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
- **`focusSidebarTree()` の改修**:
  - `appState.tabs` からアクティブなファイルパスを探してツリーを Reveal していた既存ロジックを完全に削除します。
  - `selectedElement` が存在し、それがツリー内（`elements.fileTree`）に含まれている場合は、その要素に `.focus()` を呼び出します（`focusin` リスナー連動で自動的に青色アクティブ選択になります）。
  - もし `selectedElement` が無効（まだ選択がない）な場合、ツリーの最初の要素（通常はルートフォルダ「nce」）を検索して `selectItem()` し、フォーカスを当てます。
- **新規作成（`createNewItemInTree`）確定時の処理**:
  - 新規作成が確定し、ファイルがエディタ側で開いた際、作成された新しいファイルのパスを `selectedPath` にセットします。
  - これにより、エディタにフォーカスが移った後も、ツリー側では作成された新ファイルがグレー（非アクティブ選択）でハイライトされ、次回の `Ctrl + E` の戻り先になります。

## 検証計画

### 自動テスト / コンパイルチェック
- `cargo check` および `cargo build` を実行し、ビルドエラーがないことを確認します。

### 手動検証
1. **初回 `Ctrl + E` フォーカス**:
   - アプリ起動直後（ツリー操作前）に `Ctrl + E` を押し、ツリー最上部（nce フォルダ）に青色のフォーカスが当たることを確認します。
2. **タブ切り替え時のツリー非連動**:
   - ツリーで適当なファイル（例: `A.txt`）を選択し、`Esc` でエディタへ抜けます（ツリーは `A.txt` のグレー選択を維持）。
   - エディタで別タブ（例: `B.txt`）をクリックまたは作成して切り替えます。この際、ツリーの `A.txt` の選択やスクロール位置が勝手に `B.txt` に動かないことを確認します。
3. **`Ctrl + E` のフォーカス戻り先復元**:
   - 上記の状態で再度 `Ctrl + E` を押した際、現在のアクティブタブである `B.txt` ではなく、ツリーで直前まで選択していた `A.txt` に正確に青色フォーカスが戻ることを確認します。
4. **新規作成確定時のグレー選択**:
   - `Ctrl + N` で新規ファイルを確定し、エディタにフォーカスが移った際、ツリー側では作成された新規ファイルがグレー選択表示になっていることを確認します。
