# 実装計画書 - サイドバーのキーボードナビゲーション & ファイルコピペ機能 (Phase 5)

サイドバー（ファイルエクスプローラ）において、矢印キーによるツリーの操作、エディタとの間でのキーボードによるフォーカス移動、およびファイル・フォルダの複製（コピー＆ペースト）機能を含む高度なキーボード操作を実装します。

## ユーザー確認事項
- **ファイルコピーの衝突回避ルール**: 
  複製先に同名ファイルが存在する場合、または同じフォルダ内でコピー＆ペースト（複製）を行った場合、新規作成時の仕様と同様に `ファイル名_copy` や `ファイル名_copy_01` 等の形式で自動的に連番/複製名を作成して衝突を回避します。
- **エディタフォーカスへの復帰キー**: 
  ツリーにフォーカスがある状態で `Esc` キーまたは `Enter` キー（ファイル決定時）が押された際、自動的にテキストエディタにフォーカスが戻ります。

### フロントエンド (JavaScript)

#### [MODIFY] [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
- **状態管理の追加**:
  - コピー元のパスを保持する内部変数（`copiedPath`, `copiedIsDir`）を定義します。
- **キーイベントハンドラの拡張**:
  - 各ツリーアイテム (`div.tree-item`) に `keydown` リスナーを設定し、以下の操作を処理します：
    - **`ArrowDown` / `ArrowUp`**: 現在表示されている（可視の）ツリー項目を走査し、前後へ選択を移動します。
    - **`ArrowRight`**: 選択アイテムがフォルダで閉じている場合、展開します。
    - **`ArrowLeft`**: 選択アイテムがフォルダで展開されている場合、折りたたみます。それ以外（ファイルまたは閉じているフォルダ）の場合、その親フォルダのツリーアイテムを選択します。
    - **`Enter`**: ファイルの場合は開き、エディタにフォーカスを移します。フォルダの場合は展開状態をトグルします。
    - **`Ctrl + N`**: 選択中のフォルダ配下（ファイル選択時は親フォルダ）に新規ファイルのインライン入力を開始します。
    - **`Ctrl + D`**: 選択中のフォルダ配下に新規フォルダのインライン入力を開始します。
    - **`F2`**: 選択中のアイテムの名前変更インライン入力を開始します。
    - **`Ctrl + C`**: 選択中のアイテムパスを `copiedPath` に記憶し、ステータスバーに通知を表示します。
    - **`Ctrl + V`**: 記憶された `copiedPath` を宛先フォルダ配下に複製する Tauri コマンド `copy_file_or_dir` を呼び出し、再読み込みします。
    - **`Esc`**: エディタへフォーカスを移動させます。
- **フォーカス移動 (`Ctrl + E`) の実装**:
  - `main.js` から呼び出せるように、ツリーにフォーカスを当てる関数 `focusSidebarTree()` をエクスポートします。
  - **フォーカス時の Reveal（追従）処理**:
    - 現在アクティブなタブが実ファイル（`filePath` が空ではない）の場合：ツリーからそのパスに対応するアイテムを探します。親フォルダが閉じている場合は、上位から順次フォルダを展開（`loadDirectory` の実行）し、最終的に対象ファイルを選択状態にしてスクロール表示（`scrollIntoView`）し、フォーカスを当てます。
    - 実ファイルが存在しない場合（未保存新規タブなど）：ツリーの先頭にある最初の可視アイテムにフォーカスを当てます。

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/js/main.js)
- アプリ全体の `keydown` リスナーに `Ctrl + E` のハンドリングを追加します。
- **`Ctrl + E` 押下時**:
  - デフォルトのブラウザ挙動をキャンセル（`preventDefault`）し、`sidebar.js` からエクスポートされた `focusSidebarTree()` を実行します。

## 検証計画

### 自動テスト / コンパイルチェック
- `cargo check` を実行して、Rust 側の型定義や `copy_file_or_dir` コマンドの実装にエラーがないことを確認します。
- `cargo build` を実行して、アプリケーションの正常ビルドを確認します。

### 手動検証
1. **フォーカス移動**:
   - エディタ入力中に `Ctrl + E` を押下した際、ファイルツリーの現在開いているファイルにフォーカスが移動し、ツリー上でスクロール・展開されて選択されるか確認。
   - ツリーフォーカス中に `Esc` または `Enter`（ファイル決定時）で、エディタにフォーカスが戻るか確認。
2. **キーボードナビゲーション**:
   - 矢印キー（上下）で、展開されているフォルダの中身を含めて正しく順次移動ができるか確認。
   - 矢印キー（左右）で、フォルダの展開・折りたたみ、および親フォルダへのフォーカス移動が正しく機能するか確認。
   - `Enter` でファイルが開き、かつフォーカスがエディタへ移るか確認。
3. **新規作成・名前変更ショートカット**:
   - ツリーフォーカス時に `Ctrl + N` / `Ctrl + D` で新規作成、`F2` で名前変更のインライン編集がそれぞれ開始されるか確認。
4. **コピー＆ペースト**:
   - `Ctrl + C` でファイルをコピーし、別のフォルダ（または同じフォルダ）で `Ctrl + V` を押した際に正しく複製されるか確認。
   - 同名ファイルが存在する場合、`_copy` や `_copy_01` などの連番/複製名が自动生成されて複製されるか確認。
   - ディレクトリのコピー＆ペーストで、サブフォルダや中のファイルが再帰的に複製されるか確認。
