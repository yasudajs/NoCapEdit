# 単一ウィンドウ動作（シングルインスタンス）機能の実装Walkthrough

NoCapEdit (nce) がすでに起動している場合に、新規ウィンドウを開くのではなく、起動中のウィンドウに新規タブとしてファイルを追加し、ウィンドウを最前面にフォーカスさせる「シングルインスタンス動作」を実装しました。

## 変更内容

### 1. バックエンド (Rust)
- [main.rs](file:///c:/work/NoCapEdit/src/main.rs)
  - TCPポート（`49423`）へのバインドを試みることで、1番目のインスタンスであるか2番目以降のインスタンスであるかを確実かつ迅速に判定するロジックを実装。
  - **1番目のインスタンス（プライマリ）**:
    - バインドに成功したポートで、バックグラウンドリスナー（スレッド）を起動。
    - 受信ソケットから送られてくるファイルパスを読み込み、Tauriの `single-instance-file` イベントをフロントエンドへ送信し、ウィンドウを `unminimize()` & `set_focus()` して最前面に表示。
    - ウィンドウを `tauri.conf.json` での自動生成から、 `setup` フック内での手動作成 (`tauri::WindowBuilder`) に変更し、Tauriの初期化ライフサイクルとWebView2データディレクトリ競合を根本的に回避。
  - **2番目以降のインスタンス**:
    - コマンドライン引数（`.nctx` ファイルパス等）を、ローカルTCP経由でプライマリインスタンスに送信。
    - 送信完了後、即座に `std::process::exit(0)` でプロセスを終了させるため、WebView2によるデータディレクトリのロック競合（エラー `0x800700AA`）が発生しません。

### 2. フロントエンド (JavaScript)
- [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
  - Tauri の `listen` イベントAPIを取得し、 `single-instance-file` イベントの購読（リッスン）を追加。
  - ファイルパスを受信した際、すでに同一ファイルがタブで開いていればそのタブへ切り替え、開いていなければ `openExistingFile(filePath)` を通して新規タブとして開き、アクティブタブにします。

### 3. 設定ファイル・仕様書
- [Cargo.toml](file:///c:/work/NoCapEdit/Cargo.toml)
  - バージョンを `0.1.5` に変更。
- [tauri.conf.json](file:///c:/work/NoCapEdit/tauri.conf.json)
  - `windows` リストを空配列にし、ウィンドウの手動初期化へ移行。
  - バージョンとタイトルを `0.1.5` に更新。
- [mvp-spec.md](file:///c:/work/NoCapEdit/docs/mvp-spec.md)
  - シングルインスタンス機能に関する仕様と受け入れ条件、実装履歴を追加。

## 検証結果

- `cargo build --release` により、本番向けリリースビルドが正常に作成できることを確認。
- 1つ目のプロセスを起動した状態で、2つ目のプロセスをファイル引数 `test.nctx` 付きで起動した際、WebView2のエラーが発生することなく即座にプロセスが終了し、1つ目のウィンドウで指定ファイルがタブとして追加され、フォーカスされることを実証。
