# Walkthrough: Sidebar Double Creation Bug Fix (Step 5-1-3)

キーボード操作によるツリー項目操作（新規作成、名前変更など）における重複作成バグやフォーカス遷移・選択表示の競合に関する修正を完了しました。

## 変更内容

### UI コンポーネント (JavaScript)

#### [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
1. **キーイベントのバブリング防止を追加**
   - ツリー項目（`itemDiv`）上の `keydown` イベントハンドラにおいて、自身で処理するショートカットキー（`Ctrl+N/D/C/X/V`, `ArrowKeys`, `Enter`, `Escape`, `F2`）の全てに `e.stopPropagation()` を追加しました。これにより `window` 側へのイベントバブリングを防ぎ、重複実行をブロックしました。
   - インライン入力（新規作成、名前変更）における `keydown` ハンドラでも `Escape` / `Enter` 処理時に `e.stopPropagation()` と `e.preventDefault()` を明示的に呼び出すことで、入力終了時のフォーカス喪失に起因して `Escape` イベントがバブリングし、意図せずエディタへフォーカスが戻ってしまう挙動を解消しました。

2. **新規ファイル作成完了時のツリー選択状態の連動**
   - 新規ファイル（`isDir = false`）の作成確定時、エディタへのフォーカス遷移と並行して、ツリーの DOM から新しく作成されたファイル要素を検索し、古い選択をクリア (`clearSelection()`) した上で、その新ファイルを非アクティブ選択（グレー）状態として登録する処理を追加しました。

## 検証結果

ユーザー様による実機動作確認において、以下の項目がすべて正常に動作することを確認しました。

- [x] ツリー上にフォルダがない状態で、`Ctrl+D` でフォルダ `aaa` を作成し、続けて `Ctrl+D` を押した際に「新しいフォルダ」が1つだけインライン編集状態で作成されること（二重作成バグの解消）
- [x] フォーカスがある状態で `Ctrl+N` から新規ファイルを作成した際、1つだけ「名称未設定.nctx」が作成されること
- [x] `F2` で名前変更を編集中に `Esc` でキャンセルした際、フォーカスが元のツリー項目に留まり、エディタにフォーカスが移らないこと
- [x] `Ctrl+N` で新規ファイルを作成して `Enter` で確定した際、作成された新規ファイルがエディタで開きつつ、ツリー側の選択ハイライト（非アクティブ選択のグレー）が新しいファイルへと正しく更新され、古い親フォルダ等のハイライトが消えること
- [x] その他、矢印キー移動、フォルダトグル、コピペなどのツリー基本操作が引き続き問題なく動作すること
