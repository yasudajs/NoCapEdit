# Sidebar Double Creation Bug Fix (Step 5-1-3)

ファイルツリー上の項目にフォーカスがある状態での操作（新規作成、名前変更など）におけるキーイベントのバブリング防止、および新規ファイル作成完了時におけるツリー選択状態の同期不具合を修正します。

## 提案する変更内容

### UI コンポーネント (JavaScript)

#### [MODIFY] [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)

1. **イベントバブリングのブロックを追加 (不具合1の対応を含む)**
   - `sidebar.js` の `itemDiv` に対する `keydown` リスナーにおいて、すでに処理したショートカットキー（`Ctrl+N/D/C/X/V`, `ArrowKeys`, `Enter`, `Escape`, `F2`）の全てに `e.stopPropagation()` を追加します。（※実装済み）
   - **[新規追加]** 新規作成および名前変更（`F2`）時のインライン入力欄（`<input>`）における `keydown` リスナー ([sidebar.js#L1077-L1083](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js#L1077-L1083) および [sidebar.js#L1229-L1235](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js#L1229-L1235)) の `Escape` および `Enter` 処理時に、`e.preventDefault()` と `e.stopPropagation()` を呼び出します。これにより、インプット消滅時にフォーカスが移った親要素（`itemDiv`）へ `Escape` キーイベントがバブリングして意図せずエディタへフォーカスが戻ってしまう不具合を防ぎます。

2. **新規ファイル作成完了時のツリー選択状態の同期 (不具合2の対応)**
   - `createNewItemInTree` の `commitCreate` 内において、新規ファイル（`isDir = false`）の作成成功時 ([sidebar.js#L1006-L1010](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js#L1006-L1010))、エディタへのフォーカス遷移と並行して、ツリーの DOM から新規作成されたファイルの要素を検索します。
   - 新規ファイル要素が見つかった場合、ツリー全体の古い選択状態をクリア (`clearSelection()`) し、新規ファイルを非アクティブ選択状態（`selected-inactive` クラス付与、および `selectedElement` / `selectedPath` の更新）に設定する処理を追加します。これにより、作成元の親フォルダなどに古い選択状態が残る不具合を解消します。

## 検証計画

### 手動検証手順（テスト項目）
実装完了後、以下の手動テストを行い、不具合が解消されたことと、他の操作に影響がないことを確認します。

1. **二重作成バグの解消確認**
   - ファイルツリー上にディレクトリがない状態で、`Ctrl+D` を押下してディレクトリ `aaa` を作成。
   - `aaa` にフォーカスがある状態で、再び `Ctrl+D` を押下する。
   - **期待値**: 「新しいフォルダ」が1つだけインライン編集状態（入力待ち）で作成されること。二重に作成されないこと。
2. **新規ファイルのバグ解消確認 (Ctrl+N)**
   - `aaa` にフォーカスがある状態で `Ctrl+N` を押下する。
   - **期待値**: 「名称未設定.nctx」が1つだけインライン編集状態で作成されること。
3. **F2 名前変更キャンセル時のフォーカス遷移確認 (不具合1)**
   - ツリー項目を選択し、`F2` キーを押して名前変更を開始する。
   - 名前入力欄で `Esc` キーを押してキャンセルする。
   - **期待値**: 入力がキャンセルされ、対象のツリー項目がアクティブ選択（青）されフォーカスが当たった状態が維持されること。エディタにフォーカスが移らないこと。
4. **新規ファイル作成時のツリー選択状態の確認 (不具合2)**
   - `aaa` を選択した状態で `Ctrl+N` から新規ファイルを作成し、`Enter` で確定する。
   - **期待値**: 作成された新規ファイルがエディタで開き、エディタにフォーカスが移ると同時に、ファイルツリー側では新しく作成されたファイルが非アクティブ選択（グレー）状態になり、作成元のフォルダ `aaa` の選択ハイライトが消えること。
