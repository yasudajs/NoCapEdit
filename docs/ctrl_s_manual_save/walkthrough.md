# Ctrl+S による手動上書き保存機能の確認

エディタ上で `Ctrl` + `s` キーを入力した際に、自動保存タイマーを待たずに即座にディスク上書き保存を実行する機能を実装し、バージョンを `0.1.11` にアップデートしました。

## 変更内容

### フロントエンド

#### [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
- `window.addEventListener('keydown', ...)` リスナーに `Ctrl` + `s` (または `KeyS`) のイベント検知を追加し、規定のブラウザ保存ダイアログのポップアップを抑制 (`e.preventDefault()`) しつつ手動保存処理をトリガーするようにしました。
- 新しく `triggerManualSave()` 関数を実装しました。この関数は以下の動作を行います：
  1. `syncCurrentEditorToState()` で現在のエディタに入力されたテキスト内容をタブオブジェクトに同期。
  2. 待機中の自動保存タイマー（`appState.autosaveTimer`）があれば `clearTimeout` でクリア。
  3. 保存対象を `isDirty = true` に設定し、`saveTabIfDirty()` で即座に非同期の上書き保存を実行。
  4. 保存中はステータスバーを「保存中...」に、成功後は「保存済み」に更新。

### その他
- `Cargo.toml` および `tauri.conf.json` のアプリバージョンを `0.1.11` に更新しました。

## 検証結果

- **ビルド・コンパイルチェック**: `cargo check` にてコンパイルエラーがなく、正常に `NoCapEdit v0.1.11` としてビルド可能であることを確認しました。
- **手動保存の即時性と重複防止**:
  - `Ctrl` + `s` を押した際、3秒の遅延なく即座に上書き保存処理が呼び出され、ステータスバーが更新されることをコードレベルで確認しました。
  - 手動保存と同時にタイマーが破棄（`clearTimeout`）されるため、手動保存後に自動保存が二重で走ることはありません。
