# Ctrl+S による手動上書き保存機能の実装計画

一般的なテキストエディタの操作感に合わせ、エディタ上で `Ctrl` + `s` キーを押した際に、自動保存タイマーを待たずに即座に上書き保存を実行する機能を追加します。

## ユーザーレビュー必要事項

特に大きな破壊的変更はなく、直感的な操作性を向上させるための機能追加です。

> [!NOTE]
> **仕様への影響**
> - `Ctrl` + `s` が入力された瞬間、3秒の自動保存タイマーはクリアされ、直ちにディスクへの上書き保存（安全保存方式）が実行されます。
> - 保存成功時は、ステータスバーに「保存済み」と表示されます。

## 提案する変更

### バックエンド
今回の変更はフロントエンド（JavaScript）内のキーイベントと保存トリガー処理のみであるため、バックエンド（Rust）のコード変更は発生しません（コンパイルしてバージョン `0.1.11` としてビルドするのみ）。

#### [MODIFY] [Cargo.toml](file:///c:/work/NoCapEdit/Cargo.toml)
- バージョンを `0.1.11` に更新します。

#### [MODIFY] [tauri.conf.json](file:///c:/work/NoCapEdit/tauri.conf.json)
- `package.version` を `"0.1.11"` に更新します。

### フロントエンド

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
- キーイベントリスナー（`window.addEventListener('keydown', ...)`）に `Ctrl` + `s`（または `KeyS`）の検知を追加します。
- `Ctrl` + `s` 検知時に以下を行う手動保存処理（例: `triggerManualSave()`）を呼び出します：
  1. `e.preventDefault()` でブラウザ標準の保存ダイアログのポップアップを抑制。
  2. `syncCurrentEditorToState()` を実行し、エディタに入力中の最新の文字列をアクティブタブの状態に同期。
  3. 待機中の自動保存タイマー（`appState.autosaveTimer`）があれば `clearTimeout` でクリア。
  4. アクティブなタブの `isDirty` を `true` に設定し、`saveTabIfDirty()` を直ちに非同期実行して保存。

## 検証計画

### 自動テスト
- 手動での実機動作検証を行います。

### 手動検証
- **ショートカットの動作確認**:
  - テキストを入力中、ステータスが「編集中...」の状態で `Ctrl` + `s` キーを入力し、即座にステータスバーが「保存中...」を経て「保存済み」に切り替わることを確認します。
  - 保存後、再度テキストを変更せず 3 秒待っても、自動保存の処理（「保存中...」などの表示更新）が重複して発生しないこと（タイマーが正しくクリアされていること）を確認します。
- **キーイベントのバブリング抑制**:
  - `Ctrl` + `s` 入力時に、ブラウザ標準の「ウェブページを保存」ダイアログが表示されないことを確認します。
