# Phase 5 Walkthrough - ファイルシステム監視のサイドバー部分移動

## 変更内容
1. **`src/dist/js/ui/sidebar-integration.js`**:
   - `file-system-changed` イベントリスナー（`setupSidebarFileSystemListener`）を追加。
   - 外部でのファイル・フォルダ作成/更新/削除時に、ツリーの親ディレクトリまたはルートツリーをデバウンス（250ms）を挟んで自動リロードするロジックを実装。
2. **`src/dist/js/main.js`**:
   - 不要となった `loadDirectory` や `getParentPath` の import を削除。
   - `file-system-changed` リスナーからツリー更新ループおよびデバウンス処理を削除し、タブの自動クローズ・リネーム処理のみを維持。

## 検証結果
- `cargo check`: 成功（エラーなし）
- モジュール依存関係のクリーンアップ完了

## 手動テスト手順（ユーザー検証用）
以下について動作品質の手動確認を推奨します：

- [ ] **1. 外部ファイル生成・追加**:
  - エクスプローラー等で開いているフォルダ内に新しいファイルを作成し、サイドバーツリーに自動追加されることを確認。
- [ ] **2. 外部ファイル削除**:
  - 開いているファイルを外部で削除し、ツリーから消去されること、および開いていたタブが自動的に非保存クローズされることを確認。
- [ ] **3. 外部ファイルリネーム**:
  - ファイル名を外部で変更し、ツリーと開いているタブ名が自動的に更新されることを確認。
- [ ] **4. サイドバーショートカット**:
  - `Ctrl+E`（フォーカス）、`Ctrl+N`（新規ファイル）、`Ctrl+D`（新規フォルダ）のキーボード操作が正常に作動することを確認。
