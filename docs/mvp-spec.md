## 2. プロダクト概要
NoCapEdit は、起動直後からすぐに書き始められ、ファイル名を考えずに保存できることを重視したシンプルなテキストエディタである。

一般的なテキストエディタでは、新規作成直後はまだ保存先やファイル名が決まっていないため、保存時に別途入力が必要になる。NoCapEdit ではこの手間をなくし、起動時点で保存先とファイル名を自動的に確定させることで、書くことに集中できる体験を提供する。

## 3. MVPの目的
本MVPの目的は、以下の体験をWindows上で成立させることである。
- アプリを起動した瞬間に、新規テキストファイルが自動生成されて開いている
- ユーザーはすぐに入力を開始できる
- 入力停止時、タブ切替時、タブ終了時、アプリ終了時に自動保存される
- 複数タブで新規ファイルを扱える
- 保存失敗時に、最低限のリカバリ操作ができる

## 4. 対象プラットフォームと技術方針
### 4.1 対象プラットフォーム
- 初期MVPは Windows を対象とする
- 将来的に macOS、Linux、CLI 版への展開を視野に入れる

### 4.2 開発技術
- アプリケーション基盤: Tauri
- コア実装言語: Rust
- UI方針: モダンで落ち着いた見た目を目指し、Obsidian や Zed のような雰囲気を参考にする

## 5. 想定ユーザーと利用シーン
### 5.1 想定ユーザー
- 思いついた内容をすぐに書き留めたい個人ユーザー
- ファイル名を考える操作を面倒に感じるユーザー
- シンプルな編集体験を好むユーザー

### 5.2 主な利用シーン
- メモの書き出し
- 作業ログの記録
- 下書きテキストの一時作成

## 6. MVPの対象範囲
### 6.1 MVPに含める機能
- 初回起動時のホームフォルダ設定
- 起動時の新規ファイル自動作成
- タブによる複数ファイル編集
- タブ上のプラス操作による新規ファイル作成
- CLI版との連携

## 16. 現時点の補足方針
- 初期版は一人利用を前提とする
- 秒単位衝突は通常起こりにくいが、実装上は必ず連番回避を入れる
- タブ上限は現時点では設けないが、将来必要なら警告や事実上の制限を追加できるようにする

## 17. 実装しない機能
- **タブ幅の自動縮小・調整機能（Chromeライクな挙動）**
  - タブ追加時の自動スクロールやボタンとの重なり防止など、UI制御および実装の複雑さを考慮し、初期段階では見送る。
