# タブ入力および設定機能のウォークスルー

Tabキー押下時に他要素へフォーカス移動せず、エディタにインデント（タブ文字またはスペース）を入力する機能、および設定の永続化、複数行インデント、Shift+Tabでのアンインデント機能を実装しました。

## 変更内容の要約

### フロントエンド (HTML / CSS / JS)
- **設定ダイアログの拡張**:
  - `index.html` の設定モーダル内に「Tabキーの挙動」ドロップダウンを追加。
  - `style.css` を更新し、タブ設定ドロップダウンにフォント変更セレクトボックスと同様のスタイリングを適用し、表示の崩れを防ぐためにマージンを調整。
- **設定状態の管理と保存**:
  - `main.js` 内で `tabBehavior` を `appState` に統合し、設定取得および保存処理（遅延保存含む）に組み込みました。
- **エディタ制御ロジック**:
  - `keydown` イベントで `Tab` キー押下をハンドリング。
  - 通常の Tab 入力時：単一行ではカーソル位置にインデント文字を挿入、複数行選択時はすべての行頭にインデントを追加。
  - `Shift` + `Tab` 入力時：選択範囲（単一行/複数行）の各行頭からインデントを1段階削除。
  - 選択範囲（カーソル位置）を自動調整し、テキスト挿入・削除後も選択状態を崩さず自然に編集を維持。
  - 状態変更後に `input` イベントを発火し、自動保存やメトリクス更新と完全に連動。

### バックエンド (Rust)
- `main.rs` の `AppSettings` および `SettingsResponse` に `tab_behavior` を追加。
- 初回起動時や古い設定ファイルの読み込み時にはデフォルト値として `"tab"` (タブ文字) をフォールバック値として適用する安全なマイグレーション対応を実装。
- 設定取得 (`get_settings`) と保存 (`save_settings`) コマンドで `tab_behavior` のパラメータをやり取りできるよう引数と処理を更新。

## 検証結果

### 1. ビルド検証
- `cargo build` を実行し、コンパイルが警告やエラーなく正常に完了することを確認しました。

### 2. 動作確認項目 (ユーザー検証用)
動作確認の際は、作成された実行ファイル（`target/debug/NoCapEdit.exe`）を起動し、以下の挙動を確認してください。

1. **設定の切り替えと保存**:
   - 歯車マークの設定ボタンをクリックし、「Tabキーの挙動」を「スペース 2文字」や「スペース 4文字」に切り替えて「OK」をクリック。
   - 再起動後に設定画面を開き、選択した設定が維持されていること。
2. **単一行入力**:
   - Tabキーを押下した際、フォーカスが移動せず、設定されたインデント（タブ文字またはスペース）がカーソル位置に入力されること。
3. **複数行インデント**:
   - 複数行のテキストを入力し、複数行にまたがってテキストを選択。
   - Tabキーを押下した際、選択したすべての行の先頭にインデントが追加されること。
4. **Shift + Tab インデント削除**:
   - インデントがある状態で `Shift` + `Tab` を押下した際、行頭からインデントが1段階削除されること。
   - 複数行選択した状態でも同様に、各行の先頭からインデントが削除されること。
