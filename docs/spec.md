## 2. プロダクト概要 & コアコンセプト
NoCapEdit は、起動直後からすぐに書き始められ、ファイル名を考えずに保存できることを重視したシンプルなテキストエディタである。

一般的なテキストエディタでは、新規作成直後はまだ保存先やファイル名が決まっていないため、保存時に別途入力が必要になる。NoCapEdit ではこの手間をなくし、起動時点で保存先とファイル名を自動的に確定させることで、書くことに集中できる体験を提供する。

### コア体験
- アプリを起動した瞬間に、新規テキストファイルが自動生成されて開いている
- ユーザーはすぐに入力を開始できる
- 入力停止時、タブ切替時、タブ終了時、アプリ終了時に自動保存される
- 複数タブで新規ファイルを扱える
- 保存失敗時に、最低限のリカバリ操作ができる

## 3. 対象プラットフォームと技術方針
### 3.1 対象プラットフォーム
- 現在は Windows を対象とする。
- 将来的に macOS、Linux、CLI 版への展開を視野に入れる。

### 3.2 開発技術
- アプリケーション基盤: Tauri
- コア実装言語: Rust
- UI方針: モダンで落ち着いた見た目を目指し、Obsidian や Zed のような雰囲気を参考にする。
- バージョン識別用アイコン（v0.2系のみ）:
  - ローカル環境で v0.1系 と v0.2系 を同時にインストールして使用する際、容易に見分けられるよう、v0.2系のアイコンは右下に「2」の数字が入ったデザインを使用する。
  - 対象となるリソースファイル（`icons/` 配下の png, ico, icns ファイル）に適用する。
  - **ポータブル版の実行ファイル名について**: Windowsのアイコンキャッシュの競合を防ぐため、ポータブル版（ZIP形式）に含まれる実行ファイル名は `NoCapEdit_v02.exe` にリネームして配布・利用する（インストール版のアプリ名やプログラム上のID等は競合や関連付けの破壊を防ぐため変更せず `NoCapEdit` のまま維持する）。



### 3.3 ウィンドウ表示制御（起動時ガタつき防止）
- **初期非表示（visible: false）**:
  - アプリ起動時の画面のガタつきやチラつき（Flash of Unstyled Content）を防ぐため、OSからネイティブウィンドウを作成した直後は、ウィンドウを非表示（`visible: false`）のままバックグラウンドで維持する。
- **描画完了後の表示（show）**:
  - フロントエンド側で初期化処理（設定のロード、初期テーマの適用、および初期タブのレンダリング）が完了したタイミングで、TauriのAPIを呼び出して初めてメインウィンドウを表示（`show`）する。これにより、UIの描画が完全に整った状態でウィンドウが表示される。

## 4. 機能仕様

### 4.1 初回起動とホームフォルダ
- **ホームフォルダ設定**:
  - 初回起動時にホームフォルダ設定画面を表示する。
  - デフォルトのホームフォルダは Windows の Documents 配下の nce フォルダとする。
    - 具体例: `C:/Users/<ユーザー名>/Documents/nce`
  - ユーザーはデフォルト値をそのまま採用することも、変更することもできる。
  - 設定したホームフォルダは `config.json` に永続化され、次回以降も自動的に使用される。

### 4.2 起動時・新規作成時のファイル自動生成（遅延作成）
- **遅延ファイル作成**:
  - アプリ起動時および新規タブ作成時には、ディスク上にファイルの実体（0バイトのファイル）を作成しない。
  - ファイルの作成は、ユーザーが文字を入力して最初の自動保存（または手動保存）が実行されるまで遅延させる。
  - ファイル名形式は**初回保存が実行された時刻**のローカル時刻を基準とした `yyyymmdd_hhmmss.nctx` とする。
- **連番による衝突回避**:
  - 初回保存時に同名ファイルが既に存在する場合は、`_01`、`_02` のように連番を付けて回避する。

### 4.3 タブ管理と自動クリーンアップ
- **複数タブのサポート**:
  - 複数ファイルをタブで開け、タブの表示名は以下のルールに従ってフォーマットする：
    - **ファイル未作成（未保存）状態**: タブ作成順の連番を用いて、自動保存モードでは `未保存N`、手動保存モードでは `[未保存N]` と表示する（N はアプリ起動中のセッション内でインクリメントし、再起動でリセット。上限なし）。
    - **初回保存後（自動生成ファイル）**: ファイル名形式（`yyyymmdd_hhmmss.nctx` または連番付きの `yyyymmdd_hhmmss_xx.nctx`）に一致するファイルについては、`yyyy/mm/dd hh:mm:ss`（連番付きの場合は `yyyy/mm/dd hh:mm:ss-xx`。例えば、`20260709_222843_01.nctx` は `2026/07/09 22:28:43-1` のようにインデックスを1始まりの整数値にしてハイフンで繋ぐ）として表示する。手動保存モードでは `[yyyy/mm/dd hh:mm:ss]` のように角カッコで囲む。
    - **明示的な名前のファイル**: ユーザーが明示的に保存した、または外部から開いた自動生成規則に一致しないファイルについては、ファイル名から拡張子を非表示にして表示する（例: `memo.txt` は `memo` と表示する）。
  - タブバーにあるプラス（`+`）ボタンから新規ファイルを作成して新規タブとして開く。
  - プラス（`+`）ボタンはタブ領域のすぐ右側に配置され、タブの増加に合わせて右へ移動する。タブ領域が画面幅いっぱいに達した場合は、ボタン位置は右端で固定され、タブ領域のみが横スクロール可能となるUIを採用する。
  - 新規タブ作成時やタブ切り替え時には、アクティブなタブが常に表示領域内に収まるように自動的にスクロールする。
  - **キーボードによるタブ切り替え機能**:
    - `Ctrl + Tab` キー押下で右側のタブへ切り替える。現在アクティブなタブが右端である場合は、一番左端（先頭）のタブへ切り替える（ループ）。
    - `Ctrl + Shift + Tab` キー押下で左側のタブへ切り替える。現在アクティブなタブが左端（先頭）である場合は、一番右端（最後）のタブへ切り替える（ループ）。
    - アプリ内のどの要素（エディタ、ファイルツリー、サイドバーなど）にフォーカスがある場合でも、このショートカットキーでタブを切り替えることができる。
  - 各タブは、ファイルパス、未保存フラグなどの内部状態を個別に保持する。
  - タブの切り替え時や新規作成時には、エディタに自動的にフォーカスが当たる。また、既存タブ間の切り替え時には、直前のカーソル位置（およびスクロール位置）がタブごとに保持され、切り替え後に復元される。
- **自動クリーンアップ（空ファイル削除・未作成スキップ）**:
  - タブを閉じる時、またはアプリを終了する時、対象タブのファイル名が自動生成規則（`yyyymmdd_hhmmss.nctx`）に一致し、かつ**内容が空または空白文字のみ**で構成される場合は、以下の処理を行う：
    - すでにディスクにファイルが作成されている（`filePath` が存在する）場合：そのファイルをディスクから削除してタブを閉じる。
    - まだファイルが作成されていない（`filePath` が空の）場合：ディスク操作は行わず、そのままタブを閉じる。
  - **空白文字の定義**: 空文字、半角スペース、全角スペース（`\u3000`）、タブ、改行、およびそれらの組み合わせを空とみなす。
  - 上記の削除条件に当てはまらないタブについては、未保存フラグがある場合のみ保存を試行する。

### 4.4 テキスト編集と保存仕様（自動・手動・安全保存）
- **保存モード（設定で切り替え可能）**:
  - アプリ全体の設定として「保存モード（自動保存）」と「無保存モード（手動保存）」を切り替えられる。設定変更は即座に反映され、開いている全タブに適用される。
  - **保存モード（デフォルト）**: 起動時や新規作成時にファイルを自動作成し、入力停止時やタブ切替時に自動保存する。
  - **無保存モード**:
    - 起動時や新規タブ作成時にファイルの実体を作成しない。
    - タブ名にはファイル名の代わりに `[未保存N]`（N は連番）を表示する。初回保存（`Ctrl+S`）後は `[yyyy/mm/dd hh:mm:ss]` に切り替える。
    - ステータスバー左下に `[無保存モード]` と表示する。
    - タブやウィンドウを閉じる際、未保存の変更があっても確認ダイアログは出さず、内容はすべて破棄される。
- **文字コード & 改行コード**:
  - 文字コードは UTF-8 とする。
  - 保存時の改行コードは Windows 向けとして **CRLF** に統一する。ファイル読み込み時は LF/CRLF を許容するが、保存時に正規化する。
- **自動保存（保存モード時のみ）**:
  - エディタ内のテキストが変更されたら未保存フラグを立てる。
  - 以下のタイミングで自動保存（未保存フラグがある場合のみ）を実行する。
    - ①入力停止から **3秒経過** 時（デバウンス処理）
    - ②タブ非アクティブ時（タブ切り替え時など）
    - ③タブを閉じる時
    - ④アプリ終了時
- **手動保存 (`Ctrl + S`)**:
  - `Ctrl + S` キー押下時、アクティブタブのテキスト内容を保存する。自動保存の待機時間はキャンセルされる。
  - **無保存モードでの手動保存**: ホームフォルダに対してダイアログを出さずに `yyyymmdd_hhmmss.nctx` というファイル名で直接保存を行う。保存後もタブは「無保存モード」を維持し、以降の追記は自動保存されず破棄対象となる。
- **安全保存方式**:
  - 保存時にファイル破損を防ぐため、一時ファイルに書き出してから置換する方式を採用する。これにより OneDrive 等のクラウド同期フォルダでの競合を防ぐ。
- **保存失敗時のリカバリ**:
  - 保存失敗時はステータスバーやダイアログでユーザーに通知する。
  - アプリ終了時の保存失敗では「再試行」「別名で保存」「終了を中止」の選択肢を提示する。

### 4.5 画面表示（タイトルバー・ステータスバー）
- **タイトルバーバージョン表示**:
  - ウィンドウタイトルに `NoCapEdit [ Ver X.Y.Z ]` を表示する。
  - バージョン番号は `Cargo.toml` の `version` を一元的なマスター（Source of Truth）とし、バックエンド経由でフロントエンド（`document.title`）に動的に反映する。
- **保存状態のステータス表示**:
  - 画面下部のステータスバーに、保存状態を表示する。
  - **新規作成・初期状態のステータス表示**:
    - アプリ起動時および新規タブ作成時（ファイル未作成の遅延保存待ち状態）は、現在の保存モードに応じて以下の初期ステータスを表示する：
      - **自動保存モード時**: `保存準備完了`
      - **手動保存モード時**: `[手動保存モード] ※Ctrl+Sで保存できます`
    - ユーザーが文字を入力し、最初の自動保存または手動保存が実行されて実際にディスク上にファイルが作成された時点で、初めて `[ファイル名] を作成` と表示する。
  - **編集・保存・開いた時・タブ切り替え時のステータス表示**:
    - ファイルの実体が存在する場合（既存ファイルを開いた後、または新規ファイルが初回保存された後）は、ステータスバーに `[接頭辞][ファイル名] - [状態]` の形式で表示する。
      - `[接頭辞]` : 自動保存モード時はなし。手動保存モード時は `[手動保存:Ctrl+S] ` とする。
      - `[状態]` : `編集中` / `保存中...` / `保存済み` / `保存失敗` 等（※曖昧な「準備完了」表示は行いません）。
      - 例（自動保存）: `20260711_224758.nctx - 編集中`
      - 例（手動保存）: `[手動保存:Ctrl+S] 20260711_224758.nctx - 編集中`- **キーボードナビゲーションとフォーカス制御**:
  - ファイルツリーの項目にフォーカスがある時、以下のキー操作をサポートする：
    - `ArrowDown`（↓キー）: 現在表示されている次の可視項目にフォーカスおよび選択状態を移動する。
    - `ArrowUp`（↑キー）: 現在表示されている前の可視項目にフォーカスおよび選択状態を移動する。
      - ※折りたたまれているフォルダ配下の子要素は移動対象から除外する。

    - `ArrowLeft`（←キー）: 選択中のフォルダが開いている場合は折りたたむ。閉じている、またはファイルの場合は、その親フォルダのアイテムへ選択を移動する。
    - `Enter` キー: フォルダを選択している場合は展開/折りたたみのトグル、ファイルを選択している場合はそのファイルを開く。
  - フォーカス移動ショートカット:
    - `Ctrl + E`: ファイルツリーとエディタのシームレスな行き来およびファイルの展開を行うトグルショートカット。
      - **ツリー閉・エディタフォーカス時**: ツリーを開き、ツリー側で最後に選択されていた項目にフォーカスおよび選択状態（青色）にする。未選択時は最上部項目にフォーカスする。
      - **ツリー開・エディタフォーカス時**: ツリーは開いたまま、ツリー側の項目へフォーカスを戻す。
      - **ツリー開・ツリーフォーカス時**: 
        - 選択中アイテムが**ファイル**の場合: そのファイルを開き（既に開かれている場合はそのタブをアクティブにし）、**ツリーを閉じて**エディタへフォーカスを移す。
        - 選択中アイテムが**フォルダ**の場合: **ツリーを閉じて**エディタへフォーカスを移す。
    - `Esc` (ツリーフォーカス時): **ツリーは開いた状態を維持**したまま、単にエディタへフォーカスを移す。

    | **⑥ 削除時 (Delete / Shift+Delete)** | 削除完了後、ツリーが再読込され、以下の優先順位で**自動で別のツリーアイテムを選択・フォーカス**する。<br>1. 削除したアイテムの次の兄弟要素（同じフォルダ内の直下アイテム）<br>2. なければ、前の兄弟要素<br>3. 兄弟もなく空フォルダになった場合は、その親フォルダ<br>4. ルート直下で全て消えた場合などは、ツリー先頭項目（無ければ解除） |

    | **⑧ Enterキーでのファイルオープン** | ファイルが開き、**エディタ（テキストエリア）にフォーカスが移動**する。 |
    | **⑨ 名前変更時 (F2)** | インライン入力が表示されフォーカス。<br>・**確定 (Enter) / キャンセル (Esc) 時**: 入力が終了し、**対象アイテムを選択・フォーカス（青色）した状態に戻り**、ツリー操作が継続できる。 |
    | **⑩ ツリーからの復帰 (Esc)** | ツリー操作中（切り取り中でない場合）に `Esc` キーを押した場合、**エディタ（テキストエリア）にフォーカスが移動**する。 |
    
  - **フォーカス状態の視覚表現**:
    - **アクティブ選択 (`selected` / 青色)**: ファイルツリーに操作フォーカスがある時の表示状態。
    - **非アクティブ選択 (`selected-inactive` / グレー)**: エディタ編集時など、ファイルツリーの外にフォーカスがある時の表示状態。
    - フォーカスがツリー内（`elements.fileTree`）に入った瞬間、およびツリー外（エディタ等）へ出た瞬間をブラウザイベント（`focusin` / `focusout`）で一括監視し、青/グレーの表示スタイルを常に自動で一元管理する。エディタ情報表示**:
  - ステータスバーの右側には、現在のカーソル位置、文字数、フォントサイズ、行間高さを表示する。
  - 文字数のカウントは、設定された「文字数カウント方法」に基づいて計算される。
  - 表示形式:
    - 範囲選択をしていない場合: `Ln X, Col Y | Z chars | Font xx pt | LH x xx` （Zは文字数）
    - 範囲選択をしている場合: `Ln X, Col Y | S / Z chars | Font xx pt | LH x xx` （Sは選択範囲の文字数、Zは全体文字数）

### 4.6 独自拡張子（.nctx）とOS関連付け・単一ウィンドウ動作（シングルインスタンス）
- **ファイル保存と拡張子**:
  - ファイル保存（別名保存）ダイアログでは、デフォルトを `.nctx` (NoCap Text) としつつ、`.txt`（一般テキスト）、`.ncmd`（Markdown）も選択可能とする。
- **OSファイル関連付け**:
  - インストール時に `.nctx` 拡張子をアプリに関連付け、OS上でダブルクリックした際に NoCapEdit で開けるようにする。
- **単一ウィンドウ動作（シングルインスタンス）**:
  - アプリは常に単一のメインウィンドウでのみ動作する（多重起動の防止）。
  - すでにアプリが起動している状態で、新たに `.nctx` ダブルクリック等により別プロセスで起動された場合、新規プロセスは起動引数のファイルパスを既存ウィンドウに通知して終了する。
  - 既存ウィンドウは通知されたファイルパスを受け取り、新規タブとして開きアクティブタブとする。また、ウィンドウが最小化や背後にある場合は復元し、最前面にフォーカスする。

### 4.7 ユーザー設定と永続化
ユーザー設定は設定ファイル（`config.json`）に保存され、次回起動時も自動的に適用される。

- **設定画面のUI（右ドック形式）**:
  - 設定画面は画面右側からスライドインするドック形式で表示される。
  - 設定ボタン（歯車アイコン）のクリックで開閉（トグル）でき、開いている間は設定ボタンが「✕（閉じる）」アイコンに変化する。
  - 項目が増加した場合でも、ドック内のみで縦スクロールが可能。
  - 各設定項目（セレクトボックスの変更やフォルダ選択など）は、変更された瞬間に即座に反映され、自動的に保存処理が行われる（OKボタンは廃止）。
  - 設定画面を閉じた際、エディタ（textarea）に自動的にフォーカスが戻り、開く直前のカーソル位置（およびスクロール位置）が復元される。これにより、ユーザーは設定を変更した後すぐに文字入力を再開できる。

- **テーマ切り替え**:
  - 以下の3つのカラーモードをサポートし、設定ドック内のドロップダウンで切り替える。
    - **Dark**（内部キー: `dark`）: 完全な黒ベースのダークモード（従来のデフォルト）
    - **Soft Dark**（内部キー: `soft-dark`）: グレーベースの柔らかいダークモード（Zed風）
    - **Light**（内部キー: `light`）: 白ベースのライトモード
  - DWM APIを利用し、DarkおよびSoft DarkモードではWindowsのウィンドウタイトルバーをダーク色に同期させる。LightモードではOS標準のライト色にする。
  - Windowsのタイトルバーは OSの制約上、ダーク系とライト系の2値制御のみ可能なため、DarkとSoft Darkのタイトルバー色は同一となる。
- **フォントサイズ調整 (ズーム)**:
  - 最小 `8pt`、最大 `72pt`、変更ステップ `1pt`。
  - トリガー1: `Ctrl` + マウスホイール回転
  - トリガー2: `Ctrl` + `+`（拡大） / `-`（縮小） キー
- **行間高さ調整**:
  - 最小 `1.0`、最大 `3.0`、変更ステップ `0.1`（画面上の表記は `pt` を付与）。
  - トリガー1: `Ctrl` + `Shift` + マウスホイール回転
  - トリガー2: `Ctrl` + `Shift` + `+` / `-` キー
- **フォントファミリー選択 (遅延ロード搭載)**:
  - エディタのフォントファミリーを等幅（Monospace）とその他に分類してセレクトボックスから選択できる。
  - **遅延ロード**: 起動時にはシステムフォントの全検出を行わず、設定済みの適用中フォントのみを一時表示して起動を高速化する。ユーザーがセレクトボックスをクリックした瞬間に非同期でシステムフォントの検出を開始し、全リストに更新する。
- **インデント設定 & タブ入力制御**:
  - エディタ上で `Tab` キーを押下した際、他のフォーカス移動を行わず、インデント文字を挿入する。
  - 設定画面（右ドック）から「タブ文字（`\t`）」「スペース2文字」「スペース4文字」の挙動を選択できる。
  - 複数行選択した状態で `Tab` を押下した際、選択範囲のすべての行頭にインデント文字を追加する。
  - `Shift + Tab` 押下時、選択範囲の各行頭からインデント文字を1段階分削除する。
- **文字数カウント設定**:
  - 設定画面（右ドック）から、文字数のカウント方法を「改行を含む（デフォルト）」または「文字数のみ（改行を除く）」から選択できる。
  - 設定したカウント方法は設定ファイルに永続化され、起動時や入力時に即座にステータスバーの文字数表示に反映される。


### 4.8 設定値・定数の一元管理（ハイブリッドアプローチ）
アプリケーション全体のマジックナンバーや設定値を一元管理し、保守性を高めるために以下の役割分担（ハイブリッドアプローチ）を採用する。
- **基本設定・振る舞い（Rust側で管理）**: アプリケーションの基本定数（デフォルトの保存先パス、初期テーマ設定、ファイル拡張子、ウィンドウのデフォルトサイズ、プロセス間通信のポート番号など）は、Rust側の定数モジュール（`constants.rs` 等）を「真なる情報源（Single Source of Truth）」として一元管理し、フロントエンドは起動時にTauriのAPI経由で値を取得する。
- **デザイン・UI設定（CSS変数で管理）**: フォントファミリの定義文字列（`Fira Code...` 等）やサイズ、テーマカラーなどの描画に関する値は `style.css` の `:root` に集約し、JSからはCSS変数を書き換えることでデザインを変更する設計とする。
- **フロントエンド固有のロジック（JSで管理）**: オートセーブの遅延時間（`3000ms`）やDOM要素のIDなど、OSやバックエンドに無関係なブラウザ固有の定数は、JS側の定数定義（または `config.js`）に集約する。

### 4.9 自動アップデートチェック機能
- **バックグラウンドチェック**:
  - アプリ起動直後のUI描画や初期化を妨げないよう、数秒の遅延（例：3秒程度）を挟んでからバックグラウンドで最新バージョン情報の取得を行う。
  - GitHub APIのReleasesエンドポイント（直近のリリース一覧）を利用し、起動中のアプリケーションのバージョン系統（メジャー・マイナー、例: `0.1.` または `0.2.`）に対応する最新バージョンを検索して取得する。これにより、v0.1系とv0.2系が並存する運用において、各系統に閉じた適切なアップデート通知のみが行われるように制御する。
  - ネットワークエラーやAPI制限等による取得失敗時は、エラーを握りつぶしユーザーへの通知は一切行わない。
- **通知UIの表示**:
  - 新しいバージョンが存在する場合、以下の2箇所に控えめな通知を表示する。
  1. **タイトルバー**: ウィンドウタイトルに `NoCapEdit [ Ver X.Y.Z ] (Update: vA.B.C)` の形式で追記する。
  2. **設定画面の最上段**: 設定画面（右ドック）内の最上部に、アップデート可能な旨（`アップデート可能です Ver. X.Y.Z → Ver. A.B.C`）とリリースノートへのリンクを表示する。リンククリック時は Tauri の `shell.open` API によりシステムの標準ブラウザで安全に外部ページを開く。

### 4.10 サイドバー（ファイルエクスプローラ）
- **表示とレイアウト**:
  - 画面左側にファイルエクスプローラ用のサイドバーを配置する。
  - 左端のトップバーのトグルボタンにより、サイドバーの表示・非表示を切り替えられる。
  - エディタとサイドバーの間にリサイズハンドルを設け、ドラッグにより幅を調整できる。
- **設定の永続化**:
  - サイドバーの開閉状態と設定した幅は `config.json` に永続化され、次回起動時にも復元される。
  - デフォルト値は非表示（幅 220px）とする。
- **ファイルツリー表示機能**:
  - ホームフォルダ等に設定された基準ディレクトリ（例: `nce` や `NoCapEdit_Doc` フォルダ）をルートとし、配下のファイルおよびフォルダを表示する。
  - フォルダをクリックすることで、直下の階層を動的に展開／折りたたみできるアコーディオン表示（ツリー構造）をサポートする。
  - 基準ディレクトリよりも上の階層（親ディレクトリ）へは移動・参照できないように制限する。
  - エディタで直接開けるテキスト系ファイルのみをツリーに表示する（対象拡張子: `.txt`, `.md`, `.nctx`, `.json`, `.csv`）。
  - **インデントガイドライン (ツリー線)**:
    - 展開されたフォルダの中身（子要素）の左側に、縦のガイド線（│）を表示することで、階層構造（どのファイル・フォルダがどの親フォルダに属しているか）を視覚的にわかりやすく表現する。
- **ファイル操作とコンテキストメニュー**:
  - ツリー上の要素（ファイル、フォルダ、または背景）を右クリックした際に、以下のカスタムコンテキストメニューを表示する：
    - **ファイル右クリック時**: `名前を変更`、`削除`
    - **フォルダ右クリック時**: `新規ファイル作成`、`新規フォルダ作成`、`名前を変更`、`削除`
    - **背景（何もないところ）右クリック時**: `新規ファイル作成`、`新規フォルダ作成`
  - コンテキストメニューはアプリ全体のアクティブなテーマ（Dark/Soft Dark/Light）に追従したモダンなデザインとする。
  - メニュー以外の領域をクリックするか、`Esc` キーを押した際にコンテキストメニューを閉じる。
- **新規作成および名前変更（インライン編集）**:
  - 「新規ファイル作成」「新規フォルダ作成」または「名前を変更」を選択した際、ツリーの該当箇所に一時的なテキスト入力欄（`<input>`）をインライン表示する。
  - `Enter` キー押下で確定し、`Esc` キー押下でキャンセルする。
  - 入力された名前が空の場合、新規作成時はデフォルトの名前（ファイル: `名称未設定.nctx`、フォルダ: `新しいフォルダ`。連番衝突回避を考慮する）で作成する。名前変更時は変更をキャンセル扱いとする。
  - 新規ファイルが作成された場合、自動的にそのファイルをエディタの新規タブとして開く。
- **通常削除（ゴミ箱移動とダイアログ確認）**:
  - 「削除」を選択した際、確認ダイアログ（TauriのダイアログAPI）を表示して「本当に削除しますか？」とユーザーに確認する。
  - ユーザーが承認した場合、対象のファイル・フォルダをOSのゴミ箱に移動する。
  - ただし、フォルダを削除しようとした際にそのフォルダが「空ではない」と判定された場合は、確認ダイアログを表示する前に削除処理を中断し、エラーダイアログを表示する（詳細は後述の「空ではないフォルダの削除制限」を参照）。
- **完全削除（ダイアログ確認）**:
  - ツリーの項目が選択されている状態で `Shift + Delete` キーを押した際、確認ダイアログを表示して「本当に完全に削除しますか？この操作は取り消せません。」とユーザーに確認する。
  - ユーザーが承認した場合、対象のファイル・フォルダをごみ箱を経由せずにディスクから直接削除する。
  - ただし、フォルダを削除しようとした際にそのフォルダが「空ではない」と判定された場合は、通常削除と同様に削除処理を中断し、エラーダイアログを表示する。
- **空ではないフォルダの削除制限**:
  - フォルダを削除する際（通常削除・完全削除の双方）、そのフォルダが「空ではない」と判定された場合は削除処理をブロックする。
  - **「空」の判定基準**:
    - 対象フォルダ内に、**「ツリー表示対象外の隠しファイル（ファイル名が `.` から始まるもの。例: `.DS_Store`、`.gitignore`）」および「`Thumbs.db`（大文字小文字を区別しない）」以外のファイル、またはサブフォルダが1つでも存在する場合**は「空ではない」と判定する。
    - 上記の除外対象以外のファイル（画像、PDF、テキスト、その他あらゆる一般ファイル）やサブフォルダ（たとえ空フォルダであっても）が1つでもあると、削除は拒否される。
  - **エラー時の挙動（エクスプローラ連携）**:
    - 削除がブロックされた場合、以下のメッセージを伴う確認ダイアログを表示する。
      - 「このフォルダは空ではないため削除できません。エクスプローラでフォルダを開いて中身を確認しますか？」
    - ユーザーが「はい」を選択した場合、OSのエクスプローラ（Tauriの `shell.open` API）で対象フォルダを開き、中身を確認しやすくする。
- **ツリーアイテムの選択状態とフォーカス**:
  - ツリー上の項目（ファイルやフォルダ）をマウスクリックした際、その項目を「選択状態（ハイライト）」としてマークする。
  - テキストエディタ側にフォーカスが移動した場合は、選択状態を薄いグレーなどの「非アクティブ選択状態」に変化させて画面上に残す。
  - ツリー背景の何もない部分をクリックした場合は、選択状態を解除する。
- **ドラッグ＆ドロップによる移動**:
  - ツリー内の項目をドラッグ＆ドロップして、別のフォルダへ移動できるようにする。
  - ファイルの上にドロップされた場合は、そのファイルが属する親フォルダを宛先とする。
  - フォルダの上にドロップされた場合は、そのフォルダ自体を宛先とする。
  - ツリー最下部の余白部分にドロップされた場合は、基準フォルダであるルートディレクトリを宛先とする。
  - 自分自身や自分の配下（子孫フォルダ）への移動、または現在の親フォルダへの移動（実質変化なし）など、不正な移動は処理を行わずにキャンセルし、ステータスバーに赤文字でエラーメッセージを表示（次の操作等で上書きされるまで維持）する。
  - ドラッグ時にホバーされたターゲットフォルダは、シンプルに背景色が変化するなどの視覚的なフィードバックを表示する。
- **エディタ（タブ）との連動挙動**:
  - 開いているファイルが削除された場合、そのタブを自動的に閉じる。
  - ドラッグ＆ドロップ操作によって開いているファイルが移動された場合は、該当タブを閉じることなく、タブに登録されているファイルパスを移動先の新しいパスへ自動で更新・追従させ、編集状態およびタブの表示を維持する。
  - 開いているファイルの名前が変更された場合、タブ内のファイルパスを更新し、タブの表示名も新しいファイル名に追従して更新する。
- **外部のファイル操作との連動（ファイルシステム監視）**:
  - ホームフォルダ（ルートディレクトリ）配下をバックグラウンドで常時監視（再帰的監視）する。
  - 外部のアプリケーションやOS（エクスプローラ等）によってホームフォルダ配下のファイルやフォルダが「作成」「削除」「名前変更（移動）」「更新」された場合、Tauriのカスタムイベントをフロントエンドに通知する。
  - フロントエンドは通知を受け取ると、ファイルツリーを自動で最新状態に更新する（再描画の多発を防ぐため、100〜300ms程度のデバウンス処理を挟む）。
  - また、開いているタブのファイルが外部で削除・移動された場合は、該当タブを自動的に閉じる。外部で名前変更・移動（同一フォルダ内など）された場合は、タブのパスおよび表示名を新しい名前に追従して更新する。
- **キーボードナビゲーションとフォーカス制御**:
  - ファイルツリーの項目にフォーカスがある時、以下のキー操作をサポートする：
    - `ArrowDown`（↓キー）: 現在表示されている次の可視項目にフォーカスおよび選択状態を移動する。
    - `ArrowUp`（↑キー）: 現在表示されている前の可視項目にフォーカスおよび選択状態を移動する。
      - ※折りたたまれているフォルダ配下の子要素は移動対象から除外する。
    - `ArrowRight`（→キー）: 選択中のフォルダが閉じている場合は展開する。展開している、またはファイルの場合は何もしない。
    - `ArrowLeft`（←キー）: 選択中のフォルダが開いている場合は折りたたむ。閉じている、またはファイルの場合は、その親フォルダのアイテムへ選択を移動する。
    - `Enter` キー: フォルダを選択している場合は展開/折りたたみのトグル、ファイルを選択している場合はそのファイルを開く。
  - フォーカス移動ショートカット:
    - `Ctrl + E` (エディタまたはアプリ全体の任意のフォーカス時): ファイルツリーへフォーカスを移動する。
      - 現在アクティブなタブの実ファイルが存在する場合：そのファイルをツリー上で自動展開・スクロールし、フォーカスおよび選択状態にする。
      - 開いていない、または実ファイルが存在しない場合：ツリーの先頭項目にフォーカスおよび選択状態にする。
    - `Esc` キー: テキストエディタにフォーカスを戻す。ただし、ファイルツリー上で切り取り動作中（`cut-pending`）である場合は、まず切り取り状態をキャンセル（半透明化の解除およびクリップボードのクリア）し、その後エディタにフォーカスを戻す。
    - `Enter` キー (ファイルツリーでファイルを選択してEnterで開いた際): テキストエディタにフォーカスを戻す。
- **ショートカットキーによる操作（ツリーフォーカス時）**:
  - `Ctrl + N`: 選択中のフォルダ配下（ファイル選択中の場合はその親フォルダ）に、インライン入力状態の新規ファイルを作成する。
  - `Ctrl + D`: 選択中のフォルダ配下（ファイル選択中の場合はその親フォルダ）に、インライン入力状態 of 新規フォルダを作成する。
  - `F2`: 選択中のアイテムをインライン名前変更状態にする。
  - `Ctrl + X`: 選択中のファイルまたはフォルダのパスをメモリ上に切り取り元（カット）として保持する。対象アイテムはツリー上で半透明（透明度50%）に表示される。
- **ファイルコピー・切り取り＆貼り付け（ツリーフォーカス時）**:
  - `Ctrl + C`: 選択中のファイルまたはフォルダのパスをメモリ上にコピー元として保持（コピー）する。
  - `Ctrl + V`: コピーまたは切り取りしたアイテムを選択フォルダ配下（ファイル選択中の場合はその親フォルダ）に貼り付ける。
    - **コピー（複製）時**: 対象フォルダ配下に複製を作成する。複製先に同名のアイテムが既に存在する場合、またはコピー元と同じフォルダに貼り付ける場合は、`_copy` を挟まずに初回から `_01` や `_02` などの連番を付与して衝突を自動的に回避する。
    - **切り取り（移動）時**: 対象フォルダ配下にアイテムを移動する。移動完了後、クリップボードおよび半透明表示はクリアされる（1回限り有効）。
    - 宛先に同名が存在する場合は、コピーと同様に `_01` や `_02` などの連番によって衝突を自動的に回避する。
    - コピーされたアイテムが存在しない、または複製・移動先にアクセス権がない場合はエラーをステータスバー等に表示する。
    - ディレクトリの貼り付け時は、その配下の全サブディレクトリおよびファイルを再帰的に複製・移動する。
  - **ツリー更新時の状態保持（差分更新）**:
    - 自動保存や外部ファイル監視による自動更新、または手動リロードが発生した際、ツリー全体を再生成（`innerHTML = ''`）するのではなく、変更（追加・削除・名前変更・順序変更）があった箇所のみをDOMに対してインクリメンタルに反映する「差分更新（インクリメンタル更新）」を採用する。
    - これにより、リロード時においても全体のスクロール位置や、現在選択されている要素へのフォーカス、および各フォルダの展開・折りたたみ状態（`.tree-children` の展開状態）が維持される。
    - ただし、ホームフォルダの設定変更など、ルートパスが根本から変わる場合はツリー全体を完全に初期化して再構築する。
  - **フォーカス時の選択ハイライト色調整 (Light Mode)**:
    - ライトモードにおけるファイルツリーのフォーカス対象アイテム（選択中）のハイライト色について、より優しい青色である `#CCE8FF` に変更する。



### 4.11 フロントエンドのモジュール分割（ES Modules）
単一の巨大なJSファイル（モノリス）による保守性の低下を防ぐため、フロントエンドは **ES Modules (`<script type="module">`)** を用いて機能ごとにファイルを分割・整理する。
- **グローバル汚染の防止**: `import` / `export` を用いて依存関係を明示し、グローバル変数の乱用を避ける。
- **責務の分離**:
  - `state.js` : アプリケーション全体の状態（`appState`、タブ情報など）の管理。
  - `core/` : バックエンド通信（Tauri APIラッパー）やファイル保存（自動/手動）の中核ロジック。
  - `ui/` : エディタ、サイドバー、タブバー、設定画面などの各UIコンポーネントの制御とイベントハンドリング。
  - `utils/` : アプリ全体で使い回す純粋な関数群（パス正規化など）。
- エントリポイントとして `main.js` が各モジュールを結合し、初期化処理（`init`）を実行する。

### 4.12 キーボードショートカットの制限（無効化）
- **不要なブラウザ機能の無効化**:
  - デスクトップアプリとしての操作性や安全性を確保するため、不要なブラウザ標準のキーボードショートカットを無効化する。
  - **対象とする無効化ショートカット**:
    - `Ctrl + R` / `F5` / `Ctrl + Shift + R` / `Ctrl + F5`（ページの再読み込み/リロード）
      - メモリ上の未保存状態がリセットされてデータが消えるのを防ぐため。
    - `Ctrl + P`（ブラウザの印刷機能）
      - アプリのUI要素がそのまま印刷されてしまう見栄えの悪さを防ぐため。

### 4.13 ブラウザ標準コンテキストメニューの制限
- **不要なブラウザメニューの無効化**:
  - アプリケーションとしての操作性とデザインの統一感を高めるため、リリースビルドにおいてWebブラウザ標準の右クリックコンテキストメニュー（再読み込み、印刷、ソース表示など）を無効化する。
- **デバッグ環境での例外**:
  - 開発時のデバッグ（DevToolsの起動など）を容易にするため、デバッグビルド（`cargo tauri dev` 等の `debug_assertions` 有効時）においてはコンテキストメニューを有効のまま維持する。
