# Walkthrough: 自動アップデートチェック機能の実装完了

本ドキュメントは、自動アップデートチェック機能の実装において変更された内容のまとめです。

## 変更されたファイルと内容

### 1. `src/dist/main.js`
- **変更内容**: 
  - ファイル末尾に `checkNewVersion` 非同期関数を追加しました。
  - アプリ起動時の `init` 関数内で、設定読み込み完了後に `checkNewVersion` を呼び出すようにしました。
- **詳細**:
  - `fetch` で GitHub API から最新リリース情報を取得します（起動後3秒遅延）。
  - 新バージョン検知時に、`document.title` にバージョン情報を追記し、設定画面内の通知要素にバージョン番号とリリースノートへのリンクURLをセットして表示します。

### 2. `src/dist/index.html`
- **変更内容**: 
  - 設定ダイアログ（`#settingsDialog`）の `.dialog-content` 要素の先頭に、通知表示用コンテナ（`#updateNoticeContainer`）を追加しました。
- **詳細**:
  - 初期状態は `hidden` クラスで非表示になっています。
  - `currentVerSpan`、`latestVerSpan`、およびリンク（`#releaseLink`）を含む構造を持っています。

### 3. `src/dist/style.css`
- **変更内容**: 
  - ファイル末尾に `.update-notice` コンテナおよびその内部リンク用のスタイルを追加しました。
- **詳細**:
  - 控えめで目に優しい枠線と余白、ホバー時のリンク装飾などを設定しています。

## ユーザー側での動作確認手順
1. アプリ（`cargo tauri dev` またはビルド済みバイナリ）を起動します。
2. そのままではローカルバージョンとGitHubの最新バージョンが一致（現在 0.1.16）しているため、通知は出ません。
3. （テスト用手順）`main.js` の `init` 関数内などで渡している `settings.app_version` を意図的に `"0.0.1"` など古いものに書き換えて起動してください。
4. 起動後、約3秒待つとタイトルバーに `(Update: v0.1.16)` のような通知が追記されることを確認してください。
5. 設定（歯車アイコン）を開き、最上段に `アップデート可能です Ver. 0.0.1 → Ver. 0.1.16` というメッセージとリンクが出ていることを確認してください。
6. リンクをクリックし、ブラウザで対象のGitHubリリースページが正しく開くことを確認してください。

## 追加修正 (バグ修正)
- **問題**: 設定画面の通知は表示されるが、OSのネイティブなタイトルバーが更新されない。
- **原因**: Tauri 環境では `document.title` の変更がネイティブウィンドウタイトルに自動反映されない仕様があること、および `tauri.conf.json` の `allowlist` に `window.setTitle` の権限が付与されていなかったため。
- **対応**: 
  - `main.js` にて、`document.title` を更新する際に併せて `appWindow.setTitle(newTitle)` を呼び出し、ネイティブウィンドウのタイトルバーも確実に更新されるように修正しました。
  - `tauri.conf.json` の `allowlist.window` に `"setTitle": true` の権限設定を追加しました。
