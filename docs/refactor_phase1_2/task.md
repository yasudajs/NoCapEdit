# 1-2. 手動テスト手順

本フェーズの修正（上限を9へ厳格化し、連番のゼロ埋めを廃止）が正しく動作することを確認するためのテスト手順です。

## テスト項目
- [x] 1. NoCapEditを起動し、テキストを入力して `Ctrl+S` で保存する。（例: `20260721_120000.nctx` が生成される）
- [x] 2. エクスプローラー等で、保存されたファイルと同じディレクトリに、同じ名前のファイルをもう1つ用意する（例: ダミーの空ファイルを同名で作成するなどして「同名ファイルが存在する」状況を意図的に作ります）。
- [x] 3. NoCapEdit上で新規タブ（Ctrl+N）を開き、何かテキストを入力する。
- [x] 4. 保存（Ctrl+S）を行う際、2で用意したファイルと全く同じ日時文字列を持つように、OSの時計を合わせるか、あるいは自動保存されるまでの「同一秒間」に保存を完了させる（※手動では困難な場合、既存のファイル名を「今まさに保存しようとしている時刻」にリネームして待ち構え、その時刻になった瞬間に保存するなどの工夫が必要です。難しければ、コードレビューにてゼロ埋めが廃止されたこと（`{}` への変更）をご確認いただく形でも構いません）。
- [x] 5. 保存されたファイル名が、ゼロ埋めなしの `..._1.nctx` のような形式（`_01` ではなく `_1`）で生成されることを確認する。

※上限（9）を超えた際のエラーテストは、同一秒間に10個のファイルをダミー生成する必要があり手動では困難なため、今回は「ゼロ埋めが廃止され `_1` になること」をもってロジックが新しい処理に切り替わっていることの確認とします。
