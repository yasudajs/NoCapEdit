# 選択範囲の文字数カウント機能の実装計画

エディタ内でテキストが範囲選択されている場合に、ステータスバーの文字数表示を「選択範囲の文字数 / 全体文字数 chars」に切り替えて表示する機能を追加します。範囲選択されていない場合は、従来通り「全体文字数 chars」と表示します。

## User Review Required

今回の実装にあたり、以下の仕様を前提として進めます。

> [!NOTE]
> - **改行コードの扱い**: 既存の全体文字数カウントと同様に、選択範囲の文字数についても改行文字（`\r` および `\n`）は文字数に含めず除外してカウントします。
> - **大文字・小文字表記**: ご要望に基づき、表示の末尾は既存の表記通り小文字の `chars` とします。

## Proposed Changes

### フロントエンド

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)

##### 1. [updateEditorMetrics](file:///c:/work/NoCapEdit/src/dist/main.js#L88) 関数の修正
- エディタの `selectionStart` と `selectionEnd` を取得し、それらが異なる場合に「選択範囲あり」と判定します。
- 選択範囲が存在する場合、該当部分のテキストを抽出し、改行文字を除外した上で文字数をカウントします。
- ステータスバーの `textContent` を更新する際、選択状態に応じて表示形式を切り替えます。
  - 選択なし: `Z chars`
  - 選択あり: `S / Z chars`

##### 2. [setupUIEventListeners](file:///c:/work/NoCapEdit/src/dist/main.js#L611) 関数へのイベント追加
- マウスドラッグによる選択範囲の決定直後にステータスバーの表示をリアルタイムで反映するため、`editor` 要素に対する `mouseup` イベントリスナーを追加し、`updateEditorMetrics` を実行するようにします。

## Verification Plan

### Automated Tests
- 本プロジェクトには自動テスト（単体テスト等）がセットアップされていないため、検証は手動テストにて行います。

### Manual Verification
以下のテストケースを実行し、動作確認を行います。

1. **範囲選択なしの動作確認**
   - 新規タブを開いた際、または文字を入力した際に、従来通り `0 chars` や `10 chars` のように全体文字数が正しく表示されること。
2. **マウスによるドラッグ選択時の動作確認**
   - テキストの一部をマウスでドラッグして選択した際、ステータスバーの表示が即座に `選択文字数 / 全体文字数 chars`（例: `5 / 15 chars`）に切り替わること。
   - 選択を解除（エディタの任意の位置をクリック）した際、元の `全体文字数 chars` の表示に戻ること。
3. **キーボードによる選択時の動作確認**
   - `Shift + 矢印キー` でテキストを選択または選択解除した際、リアルタイムに文字数カウントの表示が更新されること。
4. **改行を含んだ選択時の動作確認**
   - 改行を含む複数行を選択した際、改行コード（`\r\n` または `\n`）が文字数としてカウントされていないこと。
