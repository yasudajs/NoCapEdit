# [実装計画書] Ctrl+E 時のアクティブタブ自動フォーカスと親フォルダ展開

仕様書（`docs/spec.md` セクション 4.10）の規定に基づき、`Ctrl + E` 押下時に現在アクティブなタブの実ファイルをツリー上で自動検索・親フォルダ展開・フォーカス選択する機能を実装します。

## 変更内容の概要

現在 `sidebar.js` の `focusSidebarTree()` は過去にツリー上でクリック・選択された要素を優先してフォーカスしており、アクティブなタブとの連動ロジックが未実装です。
本改修により以下の動作を実現します：
1. 現在アクティブなタブオブジェクトから実ファイルパス（`filePath`）を取得する。
2. アクティブなタブの実ファイルパスが存在する場合：
   - ツリー内の `.tree-item` から該当パスを持つ要素を検索する。
   - 見つかった場合、その要素が閉じている親フォルダの中に隠れているときは、ルートまでの親 `tree-children` の `hidden` クラスを解除して可視化（親フォルダを自動展開）する。
   - 該当要素を選択（`selectItem` / `makeSelectionActive`）、フォーカス、およびスクロール表示（`scrollIntoView`）させる。
3. 実ファイルパスが存在しない（新規無保存タブなど）またはツリー外のファイルの場合：
   - ツリーの先頭項目にフォーカス・選択状態にする。

## ユーザー確認事項
- 特別な追加設定等は必要ありません。仕様書（4.10）通りの動作への修正となります。

## 変更対象ファイル

### [FrontEnd / SideBar]

#### [MODIFY] [sidebar.js](file:///c:/work/NoCapEdit/src/dist/js/ui/sidebar.js)
- `focusSidebarTree()` のロジックを修正：
  - `appState.activeTabId` および `appState.tabs` からアクティブタブの `filePath` を取得。
  - 親フォルダを非表示から表示状態にするヘルパー関数 `ensureItemVisibleAndExpanded(targetEl)` を作成。
  - **遅延ロード（Lazy Load）対応**: 外部（Windowsエクスプローラ等）から開かれたファイルのように、対象項目の `.tree-item` がまだ DOM にロードされていない場合に備え、ルートから対象ファイルまでの親フォルダ階層を順次自動で展開・ロード（`loadDirectory`）しながら DOM 上に要素を生成して探し出す関数 `expandParentFoldersToItem(targetPath)` を実装。
  - アクティブタブのファイルが存在する場合は親フォルダを自動展開してフォーカス。存在しない場合はツリー先頭へフォーカス。

## 手動検証計画

### 手動テスト項目
1. **サブフォルダ内のファイルを開いている状態で `Ctrl + E` 押下**:
   - 親フォルダが閉じていても自動展開され、対象ファイルがハイライト＆フォーカスされること。
2. **別のタブに切り替えた後に `Ctrl + E` 押下**:
   - 切り替えた新しいアクティブタブのファイルへツリー上のフォーカスが移動すること。
3. **新規未保存タブ（無保存モードの無題タブ等）を開いている状態で `Ctrl + E` 押下**:
   - ツリーの先頭項目にフォーカスされること。
4. **エディタとツリーの相互移動確認**:
   - ツリーフォーカス時に `Esc` や `Enter` でエディタへフォーカスが戻ること。
