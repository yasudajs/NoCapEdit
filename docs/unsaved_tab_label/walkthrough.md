# ウォークスルー：新規タブの「未保存」ラベル表示

「新規タブの未保存ラベル表示」および「初回保存時のタイムスタンプ付与」に関する実装が完了しました。

---

## 変更内容のまとめ

### 1. タブ名の未保存表示への変更
- 新規作成したタブについて、ディスクに書き込まれていない段階（filePathが空）では、連番つきのラベルを表示します。
  - **自動保存モード**: `未保存1`, `未保存2`, ...
  - **手動保存モード**: `[未保存1]`, `[未保存2]`, ...
- タブオブジェクトに `unsavedNumber` を保持させ、設定切り替えや一時ファイルの削除・復元が行われても一貫した連番が維持されるようにしました。

### 2. タイムスタンプ基準の修正
- ファイル名のタイムスタンプ（`yyyymmdd_hhmmss.nctx`）は、タブが作成された時刻ではなく、**最初に実際に保存（ディスク書き込み）が実行された時刻**を基準にするように変更しました。
- `saveTabIfDirty()`（自動保存時の初回保存）および `triggerManualSave()`（手動保存時の初回保存）の両方で、保存処理の実行時にタイムスタンプを動的生成して適用するよう修正しました。

### 3. 保存モード切り替え時の処理（`saveSettings`）の修正
- **手動 ➔ 自動切り替え**:
  - 内容が空（`tab.content.trim() === ''`）の未保存タブについては、ファイルを作成せず遅延作成を維持し、タブ名を `[未保存N]` から `未保存N` に変更する。
  - 内容がある未保存タブについては、自動的にファイルが作成されるように `tab.isDirty = true` に設定します。
- **自動 ➔ 手動切り替え**:
  - 空ファイルが存在する場合（`shouldDeleteEmptyFile(tab)` を満たすタブ）は、ファイルを削除して `filePath = ''` にした上で、タブ名を `[未保存N]` に書き換えるよう変更しました。

## 手動テスト確認手順

ユーザー様での動作検証の際は、以下の手順で挙動をご確認いただけますと幸いです。

1. **基本表示と連番**:
   - アプリを起動し、最初のタブが `未保存1` になっていることを確認。
   - `+` ボタンで新規タブをいくつか追加し、`未保存2`, `未保存3` と増えることを確認。
2. **自動保存時の切り替え**:
   - `未保存1` に文字を入力し、3秒待つ。
   - タブ名が `未保存1` から `yyyy/mm/dd hh:mm:ss`（保存実行時の現在時刻）に切り替わることを確認。
3. **手動保存（自動保存モード）**:
   - 新規タブを作成（例: `未保存2`）。
   - 文字を入力し、すぐに `Ctrl + S` を押下。
   - 待機時間（3秒）を待たずに即座に保存され、タブ名がタイムスタンプ名に切り替わることを確認。
4. **手動保存モードでの挙動**:
   - 設定画面で「保存モード」を「手動保存」に変更。
   - 新規タブを追加し、タブ名が `[未保存1]`（角カッコ付き連番）になることを確認。
   - 文字を入力し、`Ctrl + S` を押下。
   - タブ名が `[yyyy/mm/dd hh:mm:ss]` に切り替えられ、ホームフォルダにファイルが作成されることを確認。
5. **保存モード切り替えの挙動**:
   - 空の `[未保存1]` がある状態で設定を自動保存モードに戻すと、タブ名が `未保存1` に変わることを確認。（この時、ファイルは作成されない）
   - 文字が入力されている状態で手動モードから自動モードに変更すると、自動的に保存処理が走り、ファイルが作成され、タイムスタンプ表示になることを確認。
6. **タブのクリーンアップ**:
   - 文字を入力していない `未保存` タブを閉じると、ダイアログ表示やディスク操作を伴わずそのまま閉じることを確認。
