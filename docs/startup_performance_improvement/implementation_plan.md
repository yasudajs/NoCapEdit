# 起動時もたつき解消（フォントファミリー選択の遅延ロード化）

アプリ起動時のボトルネックとなっているシステムフォントの取得処理を遅延ロード（Lazy Load）化し、起動速度を高速化します。

## ユーザーレビュー必要事項

特に大きな破壊的変更はありませんが、以下の挙動の変化についてご確認ください。

> [!NOTE]
> **起動時およびフォント選択時のUI挙動の変化**
> - 起動時は、設定ファイルから読み込まれた「前回の適用フォント」のみがドロップダウンに表示されます。
> - ユーザーがフォント選択ドロップダウンをクリックした時点で初めてPC内の全フォントの読み込みが走るため、初回クリック時のみドロップダウンが開くまでにわずかなタイムラグ（数百ms〜1秒程度）が発生します。2回目以降 of クリックはキャッシュされるため瞬時に表示されます。

## 提案する変更

変更対象はフロントエンドのJavaScriptファイル1点のみで、バックエンド（Rust）側の変更は不要です。

### フロントエンド

#### [MODIFY] [main.js](file:///c:/work/NoCapEdit/src/dist/main.js)
- `init()` 関数内での `loadSystemFonts()` の同期的な `await` 呼び出しを削除します。
- `init()` 内で、設定ファイルから読み込んだ `appState.fontFamily` が `default` 以外である場合、それをドロップダウン（`#fontFamilySelect`）に一時的な選択肢（選択状態）として追加する処理を実装します。これにより、起動時のフォント名表示の不整合を防ぎます。
- `setupUIEventListeners()` 内で、`#fontFamilySelect` に対して `mousedown`（または `focus`）イベントを監視するリスナーを追加します。
- ロード完了状態を管理するフラグ（`isFontsLoaded`, `isFontsLoading`）を導入し、重複ロードを防ぎながら、ドロップダウン操作時に非同期で `loadSystemFonts()` を呼び出すように変更します。

## 検証計画

### 自動テスト
- 現状、TauriフロントエンドのJavaScriptに対する自動テスト（Jest等）はプロジェクトに含まれていないため、手動による実機検証を行います。

### 手動検証
- **起動速度の確認**: アプリ起動時の「もたつき（一瞬フリーズするような挙動）」が解消され、瞬時にエディタ画面が表示されることを確認します。
- **ドロップダウン初期表示の確認**: 
  - 前回の設定フォントが `default` の場合：「デフォルト (Monospace)」が選択されていること。
  - 前回の設定フォントが特定のフォント（例: `"MS Gothic"`）の場合：起動直後にドロップダウンに `"MS Gothic"` が追加され、選択状態になっていること。
- **遅延ロードの動作確認**:
  - ドロップダウンをクリックしたタイミングで、PCのシステムフォント一覧がロードされ、正しくリストに展開されること。
  - 一度展開された後は、再度クリックしても再ロードが走らないこと。
