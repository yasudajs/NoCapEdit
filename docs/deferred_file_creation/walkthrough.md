# ウォークスルー：ファイル遅延作成（Deferred File Creation）

実装計画に基づき、自動保存モードにおけるファイル実体（0バイトファイル）のディスク生成を初回保存時まで遅延させる機能を実装しました。
また、全削除時や未入力の状態でタブを閉じた際に空ファイルが余計に作成されるのを防ぐ処理も追加しました。

## 変更内容

### 1. Rustバックエンド (`src/main.rs`)
- **タイムスタンプ引数の追加**: `next_available_file_path` がフロントエンドから送られてきた `timestamp` を受け取るように変更しました。これにより、作成時のタイムスタンプをファイル名に一貫して使用できます。
- **新規コマンドの追加・旧コマンドの削除**: アトミックな新規作成＆保存を行う `create_and_save_file` を実装し、従来の即時空ファイル作成コマンド `create_auto_file` を削除しました。
- **未使用コードのクリーンアップ**: 自動生成のタイムスタンプがフロントエンド主導になったことで不要になった `chrono::Local` や `DATETIME_FORMAT` 定数を削除し、コンパイル警告を解消しました。

### 2. フロントエンド (`src/dist/main.js`)
- **新規タブ作成時**: `createNewTab` で `create_auto_file` を呼び出すのを廃止し、タブオブジェクトに `createdTimestamp` のみを記憶させ、`filePath` は空のままとしました。
- **空タブの保存スキップ**: 新規タブ作成後、入力された文字をすべて削除して空の状態で自動保存（3秒経過）またはタブが閉じられた際、ディスクへの無駄な空ファイル作成を防ぐために `saveTabIfDirty` で保存処理をスキップし、`isDirty = false` にリセットするガードを追加しました。
- **初回保存時のファイル作成**: `saveTabIfDirty` 内で `!tab.filePath` の場合に新設コマンド `create_and_save_file` を呼び出して、初めてファイルの実体を生成するように変更しました。
- **自動クリーンアップのガード**: 未作成のタブ（`filePath` が空）は、タブが閉じられる際にディスク削除処理（`delete_text_file`）をスキップするように `shouldDeleteEmptyFile` を調整しました。
